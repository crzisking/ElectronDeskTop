## 1. 整體流程

```
主進程 setInterval(5 min)
   │
   1. activeWin() 取前台視窗(標題 + appName)
   2. desktopCapturer({types:['window'], thumbnailSize:{0,0}}) 取「所有可見視窗標題清單」
      └─ thumbnailSize 0 = 不產縮圖,省 CPU,只要 name
   3. desktopCapturer({types:['screen']}) primary 螢幕 → toJPEG(70) [記憶體 Buffer]
   4. hash 比對上次,相同跳過
   5. POST /work-collect/analyze (multipart)
      payload: screenshot + activeWindow + appName + allWindows + capturedAt + userName
      resp:    category + description + confidence
   6. 截圖 Buffer 在此處生命終結(GC),桌面端零持久化
   7. WorkRecordService.insert  [只寫文字結果]
   8. webContents.send(PUSH_WORK_RECORD_NEW)
   ▼
渲染端 store 收到 push → refresh()
WorkTimelineView 更新流水線
```

**為什麼一張截圖 + 全部視窗清單**:
- 截圖只截 primary 一張(多螢幕場景投資報酬率低,先簡單做)
- `active-win` 只回前台單一視窗,多視窗並排 / 後台脈絡會丟失
- 加 `allWindows` 字串陣列(只要標題,沒縮圖)補上「同時開了什麼」的脈絡
- AI 收到「焦點 + 全部視窗 + 截圖」三層訊號互補,例如:焦點是 Chrome 但有 VSCode 開著 → AI 可判斷「查資料寫 code」而非純 browsing

**截圖端到端不落地**:
- 桌面端僅作為 HTTP body 的記憶體 Buffer,送完即釋放
- 後端處理完即丟,不寫磁碟、不寫 DB
- 任何環節都不允許 `fs.writeFile` 截圖

## 2. 採集 payload

```
POST {VITE_AI_BASE_URL}/work-collect/analyze
Content-Type: multipart/form-data
Authorization: Bearer {token}

fields:
  screenshot:    <jpeg binary>                                              一張,primary screen
  activeWindow:  "foo.ts - Visual Studio Code"                              焦點視窗標題
  appName:       "Code.exe"                                                 焦點視窗的 app
  allWindows:    ["foo.ts - VSCode","Stack Overflow - Chrome","LINE"]       全部可見視窗,JSON 字串陣列
  capturedAt:    1779155657742                                              Unix ms
```

multipart 比 base64 小 50%,選 multipart。`allWindows` 排除自家 app(避免雜訊),最多 10 個。

**沒有 `userName` 欄位** —— 使用者身份後端從 JWT `CurrentUser.UserName` 拿,不信任 client 送的值(防止偽造別人身份)。桌面端只需要帶 `Authorization: Bearer {token}` header。

## 4. SQLite Schema

`electron/main/db/schema/work-records.ts`:

```ts
import {index, integer, real, sqliteTable, text} from 'drizzle-orm/sqlite-core'

export type WorkCategory =
  | 'coding' | 'documenting' | 'browsing' | 'communicating'
  | 'meeting' | 'designing' | 'idle' | 'other'

export const workRecords = sqliteTable(
  'work_records',
  {
    id: integer('id').primaryKey({autoIncrement: true}),
    capturedAt: integer('capturedAt').notNull(),
    activeApp: text('activeApp'),
    activeWindowTitle: text('activeWindowTitle'),
    category: text('category').$type<WorkCategory>().notNull(),
    description: text('description').notNull(),
    confidence: real('confidence'),
    screenshotHash: text('screenshotHash'),   // dHash 16 hex,供閒置比對 + debug
    reason: text('reason'),                    // AI 分類理由
    // ── 集中化(見 19/20)新增 ──
    synced: integer('synced').notNull().default(0),  // 0=未同步 server,1=已同步
    syncedAt: integer('syncedAt'),                   // server 接收時間 Unix ms
  },
  (table) => ({
    idxCapturedAt: index('idx_work_capturedAt').on(table.capturedAt),
    idxSynced: index('idx_work_records_synced').on(table.synced, table.capturedAt), // 撈 unsynced
  })
)

export type WorkRecord = typeof workRecords.$inferSelect
export type NewWorkRecord = typeof workRecords.$inferInsert
```

只存 AI 分析後的文字結果,不存截圖原圖。
`synced` / `syncedAt` 是集中化後新增 —— 紀錄批次上傳到 server 後標 `synced=1`(見 19/20)。

## 5. 主進程模組

### 5.1 `electron/main/work-collector.ts`

```ts
export class WorkCollectorScheduler {
  private timer: NodeJS.Timeout | null = null
  private lastHash: string | null = null

  constructor(
    private readonly cfg: ConfigManager,
    private readonly svc: WorkRecordService,
    private readonly winMgr: WindowManager
  ) {}

  start(): void {
    const c = this.cfg.getConfig().workCollect
    if (!c?.enabled) return
    this.timer = setInterval(() => void this.tick(), (c.intervalMinutes ?? 5) * 60_000)
  }

  stop(): void {
    if (this.timer) clearInterval(this.timer)
    this.timer = null
  }

  restart(): void { this.stop(); this.start() }

  private async tick(): Promise<void> {
    try {
      const meta = await activeWin()
      if (!meta) return

      // 一張截圖 + 全部可見視窗標題清單,兩個都不落地
      const jpeg = await this.captureScreenshot()
      if (this.unchanged(jpeg)) return
      const allWindows = await this.collectAllWindowTitles()

      const result = await workCollectApi.analyze({
        screenshot: jpeg,
        activeWindow: meta.title,
        appName: meta.owner.name,
        allWindows,
        capturedAt: Date.now(),
      })

      // 只寫文字結果到 DB,截圖 Buffer 此後不再被引用
      this.svc.insert({
        capturedAt: Date.now(),
        activeApp: meta.owner.name,
        activeWindowTitle: meta.title,
        category: result.category,
        description: result.description,
        confidence: result.confidence,
      })

      this.winMgr.getMainWindow()?.webContents.send(IpcChannels.PUSH_WORK_RECORD_NEW)
    } catch (err) {
      logger.warn('採集 tick 失敗', 'WorkCollector', err)
    }
  }

  /** 擷取 primary 螢幕一張 jpeg(壓縮品質 70) */
  private async captureScreenshot(): Promise<Buffer> {
    const {width, height} = screen.getPrimaryDisplay().workAreaSize
    const sources = await desktopCapturer.getSources({
      types: ['screen'], thumbnailSize: {width, height}
    })
    return sources[0].thumbnail.toJPEG(70)
  }

  /**
   * 列出所有「可見視窗」的標題。thumbnailSize 設 0 跳過縮圖產生,只要 name 省 CPU。
   * 排除自家 app(避免雜訊),最多 10 個。
   */
  private async collectAllWindowTitles(): Promise<string[]> {
    const sources = await desktopCapturer.getSources({
      types: ['window'],
      thumbnailSize: {width: 0, height: 0}
    })
    return sources
      .map(s => s.name)
      .filter(name => name && !name.toLowerCase().includes('ichiadesktop'))
      .slice(0, 10)
  }

  private unchanged(jpeg: Buffer): boolean {
    const h = crypto.createHash('sha256').update(jpeg).digest('hex').slice(0, 32)
    if (h === this.lastHash) return true
    this.lastHash = h
    return false
  }
}
```

### 7.2 `src/stores/work-records.store.ts`

```ts
export const useWorkStore = defineStore('work', () => {
  const items = ref<WorkRecord[]>([])

  async function refresh(since: number, until: number) {
    items.value = await workApi.list(since, until)
  }

  function subscribe() {
    window.electronAPI.on(IpcChannels.PUSH_WORK_RECORD_NEW, () => {
      // 重查今日
      const start = new Date(); start.setHours(0,0,0,0)
      refresh(start.getTime(), Date.now())
    })
  }

  return {items, refresh, subscribe}
})
```

App.vue onMounted 內呼叫 `subscribe()`。

## 10. 資料邊界

公司應用,強制採集,不需要使用者確認 / 黑名單機制。唯一的硬規則:**截圖不落地**。

- 桌面端:截圖以 `Buffer` 形式只活在 `tick()` 函式作用域,送完 HTTP 後函式結束 GC 回收。**禁止任何 `fs.write*` 寫截圖**
- 桌面端 DB:`work_records` 表只有 AI 分析後的文字結果(category / description / 視窗標題),無截圖欄位
- 後端:處理完即丟,不寫磁碟、不寫 DB、不留快取
- 傳輸:強制 HTTPS

## 14. 數據視覺化(v3 — ECharts 圖表重構)

> v2 使用純 SVG 自繪（堆疊柱狀 / Donut），程式碼 ~300 行但缺乏標準圖表的特性（動畫、tooltip、響應式、匯出圖片、無障礙）。
> v3 改用 **ECharts + vue-echarts**，用宣告式配置取代手算座標，圖表能力大幅提升，同時保持體積可控（ECharts 按需引入）。

### 14.1 資料維度分析

當前 `work_records` 表的欄位及其適合的圖表型別：

| 欄位 | 型別 | 資料特徵 | 適合圖表 |
|------|------|----------|----------|
| `capturedAt` | timestamp | 時間序列，每 5 分鐘一筆，可聚合到小時/天/週 | 堆疊柱狀、面積圖、熱力圖、行事曆圖 |
| `category` | enum(8 種) | 離散類別，佔比分析 | Donut/Pie、堆疊柱狀、Treemap、雷達圖 |
| `confidence` | 0-1 float | 連續值，反映 AI 判斷可信度 | 散點圖、折線圖（時間趨勢） |
| `activeApp` | string | 應用名，可作為分組維度 | 橫向柱狀（Top N 應用排名） |
| `activeWindowTitle` | string | 標題文字，輔助資訊 | 無（只用於 tooltip / 時間軸列表） |
| `description` | string | AI 描述文字，定性資訊 | 無（只用於 tooltip / 時間軸列表） |

**核心分析維度**:
1. **時間 × 類別**: 什麼時候在做什麼 — 用堆疊柱狀
2. **類別比例**: 時間花在哪 — 用 Donut
3. **時間趨勢**: 工作效率變化 — 用折線/面積圖
4. **週模式**: 一週內哪天效率高 — 用熱力圖
5. **應用排名**: 最常用的應用 — 用橫向柱狀

---

### 14.2 技術選型: ECharts

#### 為什麼選 ECharts

| 考量 | ECharts | Chart.js | ApexCharts | 純 SVG 自繪(v2) |

| 中文生態 | ⭐⭐⭐⭐⭐ 百度開源 | ⭐⭐⭐ | ⭐⭐ | — |
| 動畫/互動 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ⭐ 手寫 |
| 按需引入 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐⭐⭐ |

### 14.4 各圖表設計詳解

#### (a) 統計卡片群 — 保留 CSS 實作

不變。4 個統計數字（今日採集數 / 覆蓋小時 / 主要類別 / 上次採集）用 Element Plus 的 `el-card` + 純 CSS，不需要圖表庫。

#### (b) 每小時堆疊柱狀 — 改用 ECharts Bar

**檔案**: `src/features/work-collect/components/HourlyStackedBar.vue`

**echarts option 核心配置**:

```typescript
// 把 records 轉成 ECharts 需要的 series 格式
function buildHourlyOption(records: WorkRecord[], startHour: number, endHour: number) {
  const hours = Array.from({ length: endHour - startHour }, (_, i) => startHour + i)

  // 每小時 × 每類別的計數矩陣
  const matrix: Record<WorkCategory, number[]> = {} as any
  for (const cat of CATEGORY_ORDER) matrix[cat] = new Array(hours.length).fill(0)

  for (const r of records) {
    const h = new Date(r.capturedAt).getHours()
    const idx = hours.indexOf(h)
    if (idx >= 0) matrix[r.category][idx]++
  }

  return {
    tooltip: {
      trigger: 'axis',
      axisPointer: { type: 'shadow' },
      formatter: (params: any[]) => {
        // params 是多 series 的 tooltip，格式化顯示各類別筆數
        const hour = params[0].axisValue
        let html = `<strong>${hour}:00 - ${hour + 1}:00</strong><br/>`
        for (const p of params) {
          if (p.value > 0) {
            html += `${p.marker} ${p.seriesName}: ${p.value} 筆<br/>`
          }
        }
        return html
      }
    },
    legend: {
      data: CATEGORY_ORDER.map(c => CATEGORY_LABEL[c]),
      top: 0,
      textStyle: { fontSize: 11 }
    },
    grid: { top: 32, bottom: 24, left: 40, right: 12 },
    xAxis: {
      type: 'category',
      data: hours.map(h => `${h}:00`),
      axisLabel: { fontSize: 10 }
    },
    yAxis: {
      type: 'value',
      minInterval: 1,
      axisLabel: { fontSize: 10 }
    },
    series: CATEGORY_ORDER.map(cat => ({
      name: CATEGORY_LABEL[cat],
      type: 'bar',
      stack: 'total',        // 堆疊
      data: matrix[cat],
      color: CATEGORY_COLOR[cat],
      emphasis: { focus: 'series' },
      itemStyle: { borderRadius: [2, 2, 0, 0] }
    }))
  }
}
```

**圖表特性**:
- 堆疊柱狀，每小時一柱
- hover 顯示 tooltip（各類別筆數）
- 圖例可點擊切換顯示/隱藏特定類別

#### (e) 每週熱力圖 — 新增

**檔案**: `src/features/work-collect/components/WeeklyHeatmap.vue`

**用途**: 展示一週七天 × 每小時的活動密度，效仿 GitHub contribution graph，直觀看出工作節奏。

```typescript
function buildWeeklyHeatmapOption(records: WorkRecord[], startHour: number, endHour: number) {
  const dayLabels = ['週一', '週二', '週三', '週四', '週五', '週六', '週日']
  const hourLabels = Array.from({ length: endHour - startHour }, (_, i) => `${startHour + i}:00`)

  // 構建 7×N 矩陣
  const data: [number, number, number][] = [] // [hourIdx, dayIdx, count]
  const matrix: number[][] = hourLabels.map(() => new Array(7).fill(0))

  for (const r of records) {
    const d = new Date(r.capturedAt)
    const dayIdx = (d.getDay() + 6) % 7 // 轉成週一=0
    const h = d.getHours()
    const hourIdx = hourLabels.findIndex(label => parseInt(label) === h)
    if (hourIdx >= 0) matrix[hourIdx][dayIdx]++
  }

  for (let hi = 0; hi < hourLabels.length; hi++) {
    for (let di = 0; di < 7; di++) {
      if (matrix[hi][di] > 0) data.push([hi, di, matrix[hi][di]])
    }
  }

  return {
    tooltip: {
      position: 'top',
      formatter: (p: any) => `${dayLabels[p.value[1]]} ${hourLabels[p.value[0]]}: ${p.value[2]} 筆`
    },
    grid: { top: 8, bottom: 24, left: 48, right: 12 },
    xAxis: {
      type: 'category',
      data: hourLabels,
      axisLabel: { fontSize: 10, rotate: 45 },
      splitArea: { show: true }
    },
    yAxis: {
      type: 'category',
      data: dayLabels,
      axisLabel: { fontSize: 11 }
    },
    visualMap: {
      min: 0,
      max: Math.max(1, ...data.map(d => d[2])),
      calculable: true,
      orient: 'horizontal',
      left: 'center',
      bottom: 0,
      inRange: { color: ['#f0f5ff', '#409EFF', '#1a3a6b'] }
    },
    series: [{
      type: 'heatmap',
      data,
      label: { show: data.length <= 70, fontSize: 9 },
      emphasis: { itemStyle: { shadowBlur: 8, shadowColor: 'rgba(0,0,0,0.3)' } }
    }]
  }
}
```

**圖表特性**:
- 7 行 × N 列格子（行=星期、列=小時）
- 顏色深度反映活動密度
- 一眼看出「週三上午最忙」「週五下午放鬆」
- 非工作日（週末）自然顯示空白

```typescript
function buildAppRankOption(records: WorkRecord[], topN: number = 5) {
  const appCounts = new Map<string, number>()
  for (const r of records) {
    if (!r.activeApp) continue
    const name = r.activeApp.replace(/\.exe$/i, '') // 去掉 .exe
    appCounts.set(name, (appCounts.get(name) ?? 0) + 1)
  }

  const sorted = [...appCounts.entries()]
    .sort((a, b) => b[1] - a[1])
    .slice(0, topN)
    .reverse() // 橫向柱狀通常從上到下

  const appColors = ['#409EFF', '#67C23A', '#E6A23C', '#F56C6C', '#A78BFA']
  return {
    tooltip: { trigger: 'axis', axisPointer: { type: 'shadow' } },
    grid: { top: 4, bottom: 4, left: 80, right: 40 },
    xAxis: { type: 'value', minInterval: 1 },
    yAxis: {
      type: 'category',
      data: sorted.map(([name]) => name),
      axisLabel: { fontSize: 11 }
    },
    series: [{
      type: 'bar',
      data: sorted.map(([_, count], i) => ({
        value: count,
        itemStyle: { color: appColors[i % appColors.length], borderRadius: [0, 4, 4, 0] }
      })),
      barMaxWidth: 24,
      label: { show: true, position: 'right', fontSize: 11 }
    }]
  }
}
```

### 14.13 風險與注意事項

1. **體積**: ECharts 按需引入約 ~300KB gzipped，是現有純 SVG 方案的唯一代價。可接受（應用已含 electron-updater 等更重依賴）。
2. **SSR**: Electron renderer 是 CSR，不受 SSR 相容性影響。
3. **Dark Mode**: ECharts 需透過 `echarts.registerTheme` 或 CSS 變數手動適配暗色模式。
4. **效能**: 每日 100 筆資料量極小，ECharts 無壓力。若後續支援月視圖(2000+ 筆)，需考慮 dataZoom 或後端聚合。
5. **遷移平滑**: 保留 TimelineList + StatCards 不動，只替換圖表元件，可漸進重構。
