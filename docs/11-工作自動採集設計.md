## 1. 整體流程

```
主進程 setInterval(5 min)
   │
   1. activeWin() 取前台視窗(標題 + appName)
   2. desktopCapturer({types:['window'], thumbnailSize:{0,0}}) 取「所有可見視窗標題清單」
      └─ thumbnailSize 0 = 不產縮圖,省 CPU,只要 name
   3. desktopCapturer({types:['screen']}) primary 螢幕 → toJPEG(70) [記憶體 Buffer]
   4. hash 比對上次,相同跳過
   5. POST /work-collect/analyze (multipart)
      payload: screenshot + activeWindow + appName + allWindows + capturedAt + userName
      resp:    category + description + confidence
   6. 截圖 Buffer 在此處生命終結(GC),桌面端零持久化
   7. WorkRecordService.insert  [只寫文字結果]
   8. webContents.send(PUSH_WORK_RECORD_NEW)
   ▼
渲染端 store 收到 push → refresh()
WorkTimelineView 更新流水線
```

**為什麼一張截圖 + 全部視窗清單**:
- 截圖只截 primary 一張(多螢幕場景投資報酬率低,先簡單做)
- `active-win` 只回前台單一視窗,多視窗並排 / 後台脈絡會丟失
- 加 `allWindows` 字串陣列(只要標題,沒縮圖)補上「同時開了什麼」的脈絡
- AI 收到「焦點 + 全部視窗 + 截圖」三層訊號互補,例如:焦點是 Chrome 但有 VSCode 開著 → AI 可判斷「查資料寫 code」而非純 browsing

**截圖端到端不落地**:
- 桌面端僅作為 HTTP body 的記憶體 Buffer,送完即釋放
- 後端處理完即丟,不寫磁碟、不寫 DB
- 任何環節都不允許 `fs.writeFile` 截圖

## 2. 採集 payload

```
POST {VITE_AI_BASE_URL}/work-collect/analyze
Content-Type: multipart/form-data
Authorization: Bearer {token}

fields:
  screenshot:    <jpeg binary>                                              一張,primary screen
  activeWindow:  "foo.ts - Visual Studio Code"                              焦點視窗標題
  appName:       "Code.exe"                                                 焦點視窗的 app
  allWindows:    ["foo.ts - VSCode","Stack Overflow - Chrome","LINE"]       全部可見視窗,JSON 字串陣列
  capturedAt:    1779155657742                                              Unix ms
```

multipart 比 base64 小 50%,選 multipart。`allWindows` 排除自家 app(避免雜訊),最多 10 個。

**沒有 `userName` 欄位** —— 使用者身份後端從 JWT `CurrentUser.UserName` 拿,不信任 client 送的值(防止偽造別人身份)。桌面端只需要帶 `Authorization: Bearer {token}` header。

## 4. SQLite Schema

`electron/main/db/schema/work-records.ts`:

```ts
import {index, integer, real, sqliteTable, text} from 'drizzle-orm/sqlite-core'

export type WorkCategory =
  | 'coding' | 'documenting' | 'browsing' | 'communicating'
  | 'meeting' | 'designing' | 'idle' | 'other'

export const workRecords = sqliteTable(
  'work_records',
  {
    id: integer('id').primaryKey({autoIncrement: true}),
    capturedAt: integer('capturedAt').notNull(),
    activeApp: text('activeApp'),
    activeWindowTitle: text('activeWindowTitle'),
    category: text('category').$type<WorkCategory>().notNull(),
    description: text('description').notNull(),
    confidence: real('confidence'),
  },
  (table) => ({
    idxCaptured: index('idx_work_captured').on(table.capturedAt),
  })
)

export type WorkRecord = typeof workRecords.$inferSelect
export type NewWorkRecord = typeof workRecords.$inferInsert
```

只存 AI 分析後的文字結果,不存截圖原圖。

## 5. 主進程模組

### 5.1 `electron/main/work-collector.ts`

```ts
export class WorkCollectorScheduler {
  private timer: NodeJS.Timeout | null = null
  private lastHash: string | null = null

  constructor(
    private readonly cfg: ConfigManager,
    private readonly svc: WorkRecordService,
    private readonly winMgr: WindowManager
  ) {}

  start(): void {
    const c = this.cfg.getConfig().workCollect
    if (!c?.enabled) return
    this.timer = setInterval(() => void this.tick(), (c.intervalMinutes ?? 5) * 60_000)
  }

  stop(): void {
    if (this.timer) clearInterval(this.timer)
    this.timer = null
  }

  restart(): void { this.stop(); this.start() }

  private async tick(): Promise<void> {
    try {
      const meta = await activeWin()
      if (!meta) return

      // 一張截圖 + 全部可見視窗標題清單,兩個都不落地
      const jpeg = await this.captureScreenshot()
      if (this.unchanged(jpeg)) return
      const allWindows = await this.collectAllWindowTitles()

      const result = await workCollectApi.analyze({
        screenshot: jpeg,
        activeWindow: meta.title,
        appName: meta.owner.name,
        allWindows,
        capturedAt: Date.now(),
      })

      // 只寫文字結果到 DB,截圖 Buffer 此後不再被引用
      this.svc.insert({
        capturedAt: Date.now(),
        activeApp: meta.owner.name,
        activeWindowTitle: meta.title,
        category: result.category,
        description: result.description,
        confidence: result.confidence,
      })

      this.winMgr.getMainWindow()?.webContents.send(IpcChannels.PUSH_WORK_RECORD_NEW)
    } catch (err) {
      logger.warn('採集 tick 失敗', 'WorkCollector', err)
    }
  }

  /** 擷取 primary 螢幕一張 jpeg(壓縮品質 70) */
  private async captureScreenshot(): Promise<Buffer> {
    const {width, height} = screen.getPrimaryDisplay().workAreaSize
    const sources = await desktopCapturer.getSources({
      types: ['screen'], thumbnailSize: {width, height}
    })
    return sources[0].thumbnail.toJPEG(70)
  }

  /**
   * 列出所有「可見視窗」的標題。thumbnailSize 設 0 跳過縮圖產生,只要 name 省 CPU。
   * 排除自家 app(避免雜訊),最多 10 個。
   */
  private async collectAllWindowTitles(): Promise<string[]> {
    const sources = await desktopCapturer.getSources({
      types: ['window'],
      thumbnailSize: {width: 0, height: 0}
    })
    return sources
      .map(s => s.name)
      .filter(name => name && !name.toLowerCase().includes('ichiadesktop'))
      .slice(0, 10)
  }

  private unchanged(jpeg: Buffer): boolean {
    const h = crypto.createHash('sha256').update(jpeg).digest('hex').slice(0, 32)
    if (h === this.lastHash) return true
    this.lastHash = h
    return false
  }
}
```

## 7. 渲染端模組

### 7.1 `src/api/modules/work-collect.api.ts`

只查本機,不直連後端:

```ts
export const workApi = {
  list: (since: number, until: number) =>
    window.electronAPI.workCollect.list({since, until}),
  export: (since: number, until: number) =>
    window.electronAPI.workCollect.export({since, until}),
}
```

### 7.2 `src/stores/work-records.store.ts`

```ts
export const useWorkStore = defineStore('work', () => {
  const items = ref<WorkRecord[]>([])

  async function refresh(since: number, until: number) {
    items.value = await workApi.list(since, until)
  }

  function subscribe() {
    window.electronAPI.on(IpcChannels.PUSH_WORK_RECORD_NEW, () => {
      // 重查今日
      const start = new Date(); start.setHours(0,0,0,0)
      refresh(start.getTime(), Date.now())
    })
  }

  return {items, refresh, subscribe}
})
```

App.vue onMounted 內呼叫 `subscribe()`。

## 10. 資料邊界

公司應用,強制採集,不需要使用者確認 / 黑名單機制。唯一的硬規則:**截圖不落地**。

- 桌面端:截圖以 `Buffer` 形式只活在 `tick()` 函式作用域,送完 HTTP 後函式結束 GC 回收。**禁止任何 `fs.write*` 寫截圖**
- 桌面端 DB:`work_records` 表只有 AI 分析後的文字結果(category / description / 視窗標題),無截圖欄位
- 後端:處理完即丟,不寫磁碟、不寫 DB、不留快取
- 傳輸:強制 HTTPS

## 11. 錯誤處理

| 情況 | 行為 |
|---|---|
| 後端接口失敗 / 超時 | logger.warn,本筆不寫 DB |
| 後端 401 | 走既有 auth.interceptor(AD 重登 / /login) |
| 截圖擷取失敗(權限拒絕) | logger.error 並停止後續 tick;主進程記下狀態,下次 App 重啟再試 |
| activeWin 失敗 | logger.warn,該 tick 跳過 |
| 連續同畫面(hash 相同) | skip,不上傳 |
