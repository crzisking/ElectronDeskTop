## 2. 三條鐵律

### 鐵律一:錄入永遠只有一句話

- 全域熱鍵 **`Ctrl + /`**(Electron `globalShortcut` 全域註冊,本應用運行時優先;非系統快捷鍵,不衝突)→ 彈**單行輸入框**:
  一句 placeholder `記錄點什麼...`,Enter 存、Esc 關,2 秒結束。
- 複用現有 `electron/main/idea-capture/hotkey-manager.ts` 的全域熱鍵設施(靈感速記已有)。
- 語音錄入:**借 Windows 11 系統語音輸入(Win+H)**,不自建 STT。Win+H 會把辨識文字打進**當前聚焦的輸入框**,
  所以錄入框只要是正常可聚焦 input,語音與手打天然共用同一個框。
  流程:`Ctrl+/` → 彈錄入窗並聚焦 input → **確認前台聚焦後**程序化觸發一次 Win+H(user32 `keybd_event` 合成 `LWin+H`,
  免重型原生依賴)。**護欄(避免問題)**:①先聚焦再觸發(否則語音打進別的 app);②只發一次+去抖(Win+H 是開關鍵);
  ③語音尽力而為,失敗靜默、手打不受影響;④配置「錄入時自動開語音」可關,框內另放 🎤 手動觸發;⑤關框不強行關語音面板。
  排 **P4**;P3 錄入窗先建成「可聚焦 input + 預留 🎤 觸發點」。

### 鐵律二:錄入後 AI 在後台工作

- 存下的**永遠是那句原話**(`content`);AI 是**異步增強**,絕不阻塞錄入,也絕非前置依賴。
- 複用靈感速記 `refiner.ts` 的**單 worker 後台佇列**模式:完成後 push 回 dock 就地刷新。
- AI 產出:`kind`(任務/Bug/會議/提醒)、`title`、`priority`、`owner`、截止。
- **相對時間交給代碼算**:prompt 把「今天」給 LLM,LLM 只回相對詞(`下午`/`周五`)+ `timeOfDay`,由代碼換算成絕對 `dueAt`
  (`下午→今天 14:00`、`周五→本周五`)。日期數學不交給 LLM,避免算錯。

### 鐵律三:編輯是卡片,不是表單

- **不要**一堆空輸入框(標題□ 描述□ 截止□ 標籤□ —— 沒人會填)。
- 點開任務 → 一張**卡**:標題 + 幾個可點 **chip**(`截止` / `優先級` / `+備注`),點 chip 就地改(循環切換 / 迷你選擇器)。

## 3. 桌面呈現(核心,使用者最在意)

**位置:螢幕頂部**(預設主螢幕;多螢幕只放主屏)。滑鼠移到**螢幕最上緣** → 面板**下滑**現身;移開 → 收回。
**呈現:卡片、可掃,絕不做成表單。**

**技術做法**(定案:**固定尺寸透明穿透窗 + CSS 動畫**):一個**透明、無邊框、置頂、不進工作列、focusable:false、
固定滿寬固定高(~480px)**的獨立窗口,貼主螢幕頂緣。

- 平時 `setIgnoreMouseEvents(true,{forward:true})` → **全透明看不見 + 點擊穿透**(不擋任何東西,含右上角視窗按鈕),
  但 forward 讓 renderer 仍收得到 mousemove。
- renderer 用**整窗 mousemove** 偵測(不靠元素 hover):滑到頂部正中一小段 → 送 `TODO_DOCK_SET_INTERACTIVE(true)`
  轉為可交互,面板用 **CSS transform 平滑滑下**;滑鼠離開面板矩形 → 送 false 恢復穿透、面板滑回。
- 動畫在 CSS(窗口不縮放)→ 連續、無 px 尺寸提示、內容完整;平時不 render 任何可見物 → 純透明。
- 只在滑鼠位於面板上才 interactive → 右上角視窗按鈕永不被擋。

**視覺語言 = 桌面小組件(widget),不是清單 / 表單。核心原則:一眼看到重點、顯示與操作分開。**

**收起態 = 資訊條(不是乾巴巴一個數字)**
頂緣一條窄 bar,**直接把最緊急那件亮出來**(🔥 標題 · 時間)+ 右側計數(`今天 N · 收件箱 M`)+ 唯一的 ⌄ 展開把手。
**不悬停也能掃到最重要的**,條上不放其它按鈕。(可選:多條輪播「滾動顯示」。)

**展開態 = 兩個分開的區**

1. **操作區(頂部一行,和卡片分開)**:左邊日期標題「今天 · 周三」,右邊**集中放按鈕** —— `+ 快速記錄(Ctrl /)`、`查看全部(N)`、(
   後續)篩選。所有動作歸這裡,不塞進卡片。
2. **顯示區 = 橫向卡片帶(可左右滾),只放資訊**:**按重要度排序**(逾期 → 今天按時間 → 高優 → 其餘 → 收件箱)。
    - **第一緊急 = 放大的「焦點卡」**(更寬、標題 18px、紅框、標「最緊急」)—— 真正做到一眼鎖定重點。
    - 常規卡:類型標(帶色 chip)+ 優先級(用色不用字段框)+ **標題為主** + 截止時間**用顏色編碼緊急度**(
      逾期紅/今天橙/之後灰)。
    - **每張卡只留一個安靜的 ○ 完成圈**(最高頻動作,懸停變 ✓);**次級操作(延後/改時間/優先級/備注)點開卡片後在下方獨立「詳情區」出現
      **,不散落在卡面。
    - 收件箱卡:虛線框「AI 整理中」,列未歸類項。

**完整看板(查看全部)**:較大視圖,分區 今天 / 本週 / 收件箱,用於整理與批量處理。

> 顏色只編碼「緊急度 / 類型」兩件事,不堆色。編輯就地改(§鐵律三),不彈表單。
> 備選皮膚(P5):Mac widget 單卡流。主模式是橫向卡片帶。

## 4. 收件箱(Inbox)+ 漸進式完善

  使用者**不用手動整理**。
- **漸進式完善**(殺手鐧):第一次只存「ERP權限」;隔天懸停面板時,該條帶一個**極輕提示**:
  「昨天記的『ERP權限』,補個時間? `[今天][明天][不用]`」,一點即完成。
  用 `enrichPromptedAt` 節流:僅在缺關鍵欄位(截止/優先級)且隔天後,輕提一次,不反覆煩。

## 5. 資料模型(**本地 SQLite / Drizzle 新 feature** `todos`)

新表 `electron/main/db/features/todos/schema.ts`,時間一律 unix ms:

| 欄位                                    | 說明                                            |
|---------------------------------------|-----------------------------------------------|
| `id`                                  | 主鍵(nanoid / uuid)                             |
| `content`                             | **那句原話**(永遠保留)                                |
| `title`                               | 顯示標題;缺省 = content,AI 可提煉                      |
| `note`                                | 備注(漸進補充)                                      |
| `status`                              | `inbox / active / done / dropped`             |
| `kind`                                | `task / bug / meeting / reminder`(AI;預設 task) |
| `priority`                            | `0 low / 1 mid / 2 high`(對齊後端語義)              |
| `category`                            | `work / life`(AI)                             |
| `owner`                               | 負責人(AI)                                       |
| `dueAt`                               | 絕對截止 ms(可空)                                   |
| `dueKind`                             | `today / thisweek / none`(顯示用)                |
| `source`                              | `keyboard / voice`                            |
| `aiState`                             | `pending / done / failed / skipped`(後台分析狀態)   |
| `enrichPromptedAt`                    | 漸進式完善節流(ms)                                   |
| `createdAt / updatedAt / completedAt` | —                                             |

一支 migration(`npm run db:generate`)。**一次性導入**:首啟時可 best-effort 從後端 `Project_Memos` 拉現有備忘錄 → 本地
seed
(離線失敗不致命,不阻塞)。
