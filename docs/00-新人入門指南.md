## 1. 這個項目是什麼？

這是一個**企業桌面客戶端**，用 Electron 打包成 Windows 應用程式（.exe）。

功能：
- **統一平台**：把公司所有內部系統（ERP、BI、OA 等）集中在一個入口，點一下就能打開
- **AI 快捷功能**：各種 AI 工具入口（後續自行開發）
- **業務安排與尋找**：維護業務流水線、搜索業務負責人
- **浮球**：最小化後在屏幕邊緣懸浮，隨時快速打開

---

## 2. Electron 是什麼？為什麼用它？

**Electron = Node.js（後端能力）+ Chromium（網頁渲染）打包成桌面應用**

你用 HTML/CSS/JavaScript 寫界面，Electron 幫你打包成一個可以在 Windows/Mac 上雙擊運行的 .exe 文件。

類比：
- 普通網頁：只能在瀏覽器裡跑，不能讀寫本地文件
- Electron 應用：既能用網頁技術做界面，又能調用 Node.js 讀寫文件、操作系統托盤等

## 3. 三個進程——最核心的概念

這是理解 Electron 項目最重要的一件事。

```
┌──────────────────────────────────────────────────────────────────┐
│                         你的 Electron 應用                        │
│                                                                  │
│  ┌─────────────────┐      ┌──────────────┐     ┌─────────────┐  │
│  │   主進程 (Main)  │      │  預加載腳本   │     │ 渲染進程    │  │
│  │                 │◄────►│  (Preload)   │◄───►│ (Renderer)  │  │
│  │ electron/main/  │      │electron/     │     │ src/        │  │
│  │ index.ts        │      │preload/      │     │ *.vue       │  │
│  │                 │      │index.ts      │     │             │  │
│  │ Node.js 環境    │      │ 橋樑/安全層  │     │ Chromium    │  │
│  │ 可以讀寫文件    │      │              │     │ 就是網頁    │  │
│  └─────────────────┘      └──────────────┘     └─────────────┘  │
└──────────────────────────────────────────────────────────────────┘
```

### 主進程 (`electron/main/`)
- **是什麼**：整個應用的「大腦」，Node.js 環境
- **能做什麼**：創建窗口、讀寫文件、調用操作系統 API（托盤圖標、通知）
- **不能做什麼**：直接操作網頁 DOM
- **關鍵文件**：`electron/main/index.ts`（入口）

### 渲染進程 (`src/`)
- **是什麼**：顯示界面的「網頁」，Chromium 瀏覽器環境
- **能做什麼**：渲染 HTML/CSS/Vue 組件，響應用戶點擊
- **不能做什麼**：直接讀寫本地文件（安全限制）
- **關鍵文件**：`src/main.ts`（入口）、`src/App.vue`（根組件）

### 預加載腳本 (`electron/preload/`)
- **是什麼**：主進程和渲染進程之間的「安全橋樑」
- **作用**：把主進程的能力「安全地」暴露給網頁，網頁通過 `window.electronAPI.xxx()` 調用
- **為什麼需要它**：直接讓網頁調用 Node.js 太危險（XSS 攻擊），所以要白名單過濾
- **關鍵文件**：`electron/preload/index.ts`（主窗口橋樑）

### 它們怎麼互相通信？

```
渲染進程                    主進程
(Vue 頁面)                 (Node.js)
    │                          │
    │  window.electronAPI      │
    │  .window.minimize()      │
    │─────────────────────────►│  ipcMain.on('window:minimize', ...)
    │                          │  win.minimize()
    │                          │
    │◄─────────────────────────│  win.webContents.send('push:xxx', data)
    │  window.electronAPI      │
    │  .on('push:xxx', cb)     │
```

## 5. 代碼是怎麼跑起來的？啟動流程

當你運行 `npm run dev`，發生了以下事情：

```
npm run dev
    │
    ▼
electron-vite dev
    │
    ├─ 1. 編譯主進程代碼  electron/main/index.ts → out/main/index.js
    │
    ├─ 2. 編譯預加載腳本  electron/preload/index.ts → out/preload/index.js
    │
    ├─ 3. 啟動 Vite 開發服務器  http://localhost:5173/
    │     負責把 src/ 下的 Vue 文件實時編譯成網頁
    │
    └─ 4. 啟動 Electron
          │
          ▼
       electron/main/index.ts 開始執行
          │
          ├─ new ConfigManager().load()     ← 讀取 config/app-config.json
          ├─ new WindowManager()            ← 準備窗口管理器
          ├─ windowManager.createMainWindow()  ← 創建主窗口
          │     └─ 加載 http://localhost:5173/  ← 就是 Vue 的界面
          ├─ windowManager.createFloatingBallWindow()  ← 創建浮球窗口（默認隱藏）
          ├─ new FloatingBallManager()      ← 準備浮球拖動邏輯
          ├─ registerAllHandlers()          ← 注冊所有 IPC 消息監聽器
          └─ new TrayManager()             ← 創建托盤圖標
```

同時，Vue 應用也在啟動：

```
src/main.ts 開始執行
    │
    ├─ createApp(App)          ← 創建 Vue 應用
    ├─ app.use(Pinia)          ← 注冊狀態管理
    ├─ app.use(Router)         ← 注冊路由
    ├─ app.use(ElementPlus)    ← 注冊 UI 組件庫
    ├─ 注冊所有圖標            ← 側邊欄動態圖標需要
    └─ app.mount('#app')       ← 掛載到 index.html 的 <div id="app">
          │
          ▼
       src/App.vue 的 onMounted() 執行
          │
          ├─ configStore.loadConfig()   ← 通過 IPC 向主進程請求配置
          ├─ authStore.restoreSession() ← 嘗試恢復登錄狀態
          └─ 注冊主進程推送事件監聽
```

## 6. 配置文件驅動所有菜單

**這個項目的設計原則：不改代碼，只改配置文件就能改菜單。**

### `config/app-config.json` 控制什麼？

```json
{
  "sidebar": {
    "items": [           ← ① 左側導航菜單項
      { "id": "...", "label": "統一平台", "icon": "Grid", "routeName": "unified-platform" }
    ]
  },
  "floatingBall": {
    "quickMenu": [       ← ② 浮球右鍵菜單項
      { "id": "...", "label": "打開主窗口", "action": { "type": "show-main-window" } }
    ]
  },
  "unifiedPlatform": {
    "systems": [         ← ③ 統一平台的系統卡片
      { "id": "sys-erp", "name": "新ERP 系統", "url": "http://..." }
    ]
  },
  "aiQuickFunctions": {
    "tools": [           ← ④ AI 功能的工具卡片
      { "id": "qanda", "name": "AI 問答", "routeName": "ai-qanda" }
    ]
  }
}
```

### 配置是如何流動到界面的？

```
config/app-config.json
        │
        │ 主進程啟動時讀取
        ▼
electron/main/config-manager.ts
        │
        │ 渲染進程通過 IPC 請求（window.electronAPI.config.read()）
        ▼
src/stores/config.store.ts  ← Pinia Store，存在內存裡
        │
        │ Vue 組件從 Store 讀取（const configStore = useConfigStore()）
        ▼
src/components/layout/SidebarNav.vue    ← 讀 configStore.sidebarItems
src/views/UnifiedPlatform/...View.vue   ← 讀 configStore.platformSystems
src/views/AiQuickFunctions/...View.vue  ← 讀 configStore.aiConfig.tools
```

### 我要加一個側邊欄菜單項，只需要：

1. 在 `app-config.json` 的 `sidebar.items` 加一條（圖標名稱要用 Element Plus 的圖標名）
2. 在 `src/router/index.ts` 加對應的路由
3. 在 `src/views/` 新建頁面組件

就這三步，不用動任何其他代碼。

## 7. Vue 組件：頁面是怎麼搭起來的

### 組件樹（應用打開後的結構）

```
App.vue（根，負責初始化）
  └─ <router-view>（根據 URL 渲染不同東西）
       └─ AppLayout.vue（帶佈局的父路由組件）
            ├─ TitleBar.vue（頂部標題欄，有拖動區）
            └─ <div class="app-body">
                 ├─ SidebarNav.vue（左側導航）
                 │    └─ SidebarNavItem.vue × N（每個菜單項）
                 └─ <main>（右側內容區）
                      └─ <router-view>（渲染當前頁面）
                           ├─ UnifiedPlatformView.vue  （/unified-platform）
                           ├─ AiQuickFunctionsView.vue  （/ai-quick-functions）
                           └─ BusinessView.vue          （/business）
```

### 一個 Vue 組件長什麼樣？

```vue
<!-- src/views/UnifiedPlatform/SystemCard.vue 為例 -->

<script setup lang="ts">
// ① 這裡寫 JavaScript/TypeScript 邏輯
// setup 是 Vue 3 Composition API，讓代碼更好組織

import { computed } from 'vue'                  // Vue 的工具函數
import type { SystemLink } from '@/types/...'   // TypeScript 類型

// defineProps：聲明這個組件接收什麼數據（從父組件傳入）
const props = defineProps<{ system: SystemLink }>()

// defineEmits：聲明這個組件會發出什麼事件（通知父組件）
const emit = defineEmits<{ (e: 'open', system: SystemLink): void }>()

// computed：根據其他數據自動計算的值
const firstLetter = computed(() => props.system.name[0].toUpperCase())
</script>

<template>
  <!-- ② 這裡寫 HTML 模板，{{ }} 輸出數據，v-if/v-for 控制渲染 -->
  <div class="card" @click="emit('open', props.system)">
    <span>{{ firstLetter }}</span>
    <h3>{{ props.system.name }}</h3>
  </div>
</template>

<style scoped>
/* ③ 這裡寫 CSS，scoped 表示只對本組件生效 */
.card { cursor: pointer; }
</style>
```

### `@/` 是什麼意思？

`@/` 是 `src/` 目錄的別名，配置在 `electron.vite.config.ts`：

```typescript
'@': resolve('src')   // @ 等於 src/ 文件夾
```

所以 `import xxx from '@/stores/config.store'` 等於 `import xxx from 'src/stores/config.store'`。

## 8. Pinia Store：共享數據放哪裡

**問題：A 組件改了數據，B 組件怎麼知道？**

答：把數據放在 Store（全局狀態），所有組件都從同一個地方讀。

本項目有三個 Store：

### `config.store.ts` — 應用配置
```typescript
// 在任意組件中使用：
import { useConfigStore } from '@/stores/config.store'
const configStore = useConfigStore()

// 讀取數據（這是響應式的，配置變了界面自動更新）
configStore.sidebarItems        // 側邊欄菜單項
configStore.platformSystems     // 統一平台的系統列表
configStore.aiConfig?.tools     // AI 工具列表
configStore.businessConfig      // 業務安排與尋找配置
```

### `auth.store.ts` — 登錄狀態
```typescript
import { useAuthStore } from '@/stores/auth.store'
const authStore = useAuthStore()

authStore.accessToken           // 當前登錄的 Token（string | null，僅內存）
authStore.isAuthenticated       // 是否已登錄（boolean）
authStore.user                  // 用戶資訊（UserProfile | null）
authStore.isRestoringSession    // App 啟動恢復會話中（boolean，路由守衛用到）
authStore.login(user, pwd)      // 登錄（LoginView 的 handleLogin 使用）
authStore.logout()              // 登出（清除 token + 狀態，路由守衛自動跳轉 /login）
```

### `ui.store.ts` — 界面狀態
```typescript
import { useUiStore } from '@/stores/ui.store'
const uiStore = useUiStore()

uiStore.sidebarCollapsed        // 側邊欄是否折疊
uiStore.isWindowMaximized       // 窗口是否最大化
uiStore.toggleSidebar()         // 切換側邊欄
```

## 9. IPC 通信：頁面和 Electron 怎麼說話

### 渲染進程 → 主進程（Vue 頁面發消息給 Electron）

所有可用方法都在 `window.electronAPI`（由預加載腳本注入）：

```typescript
// 窗口控制
window.electronAPI.window.minimize()    // 最小化
window.electronAPI.window.maximize()   // 最大化/還原
window.electronAPI.window.close()      // 關閉（其實是隱藏，變成浮球）

// 配置
const config = await window.electronAPI.config.read()   // 讀取配置

// Token
const token = await window.electronAPI.auth.getToken()  // 讀取登錄 Token
await window.electronAPI.auth.setToken('xxx')           // 存儲 Token

// 浮球
window.electronAPI.floatingBall.startDrag()             // 開始拖動
window.electronAPI.showContextMenu()                    // 顯示右鍵菜單
```

### 主進程 → 渲染進程（Electron 推送消息給 Vue 頁面）

```typescript
// 在 Vue 組件的 onMounted 中監聽：
window.electronAPI.on('push:window-maximized', (...args) => {
  const isMaximized = args[0] as boolean
  // 更新界面...
})

// 記得在 onUnmounted 中取消監聽：
window.electronAPI.off('push:window-maximized', callback)
```

### IPC 頻道名稱在哪裡定義？

所有頻道名稱在 `electron/shared/ipc-channels.ts`，這是主進程和渲染進程共用的常量：

```typescript
export const IpcChannels = {
  WINDOW_MINIMIZE: 'window:minimize',  // 不要直接用字符串，用常量
  CONFIG_READ: 'config:read',
  // ...
}
```

### 添加新的 IPC 通信，需要改哪些文件？

假設你要添加「獲取系統版本號」功能：

```
1. electron/shared/ipc-channels.ts
   → 添加: APP_GET_VERSION: 'app:get-version'

2. electron/preload/index.ts
   → 暴露: getVersion: () => ipcRenderer.invoke('app:get-version')

3. electron/main/ipc-handlers/index.ts（或新建一個 handlers 文件）
   → 添加: ipcMain.handle('app:get-version', () => app.getVersion())

4. src/types/electron.d.ts
   → 在 electronAPI 類型裡添加: getVersion: () => Promise<string>

5. 在 Vue 組件裡使用:
   → const version = await window.electronAPI.getVersion()
```

## 11. 浮球是怎麼實現的

### 為什麼浮球是獨立的？

浮球是**第二個 BrowserWindow**（獨立窗口），它和主窗口完全分離。

```
主窗口 BrowserWindow          浮球 BrowserWindow
──────────────────            ──────────────────
frame: false                  frame: false
1200 x 800                    60 x 60
普通窗口                      transparent: true（透明）
                              alwaysOnTop: true（置頂）
                              skipTaskbar: true（不在任務欄）
```

圓形效果：窗口是矩形的，但因為透明背景，只有 CSS 的 `border-radius: 50%` 圓形部分可見。

### 為什麼拖動不用 `-webkit-app-region: drag`？

因為 `-webkit-app-region: drag` 會**吞掉所有鼠標事件**（點擊失效）。

所以我們用了另一種方式：
1. `mousedown` → 告訴主進程「開始拖動」
2. 主進程每 16ms 輪詢鼠標位置，更新窗口位置（60fps）
3. `mouseup` → 告訴主進程「停止拖動」，觸發吸附動畫

### 為什麼右鍵菜單用原生菜單？

浮球窗口只有 80×80px，如果在裡面渲染菜單會被裁剪。所以：
- 右鍵 → 通過 IPC 告訴主進程 → 主進程用 `Menu.popup()` 彈出原生菜單
- 原生菜單在操作系統層面顯示，不受窗口大小限制

## 13. 常見問題 FAQ

**Q：修改了 app-config.json，但界面沒變？**
A：需要重啟應用（`npm run dev`）。配置只在啟動時讀取一次，除非開啟了熱更新。

**Q：圖標不顯示怎麼辦？**
A：圖標名稱必須和 [Element Plus Icons](https://element-plus.org/zh-CN/component/icon.html) 完全一致，區分大小寫。例如 `"Grid"`、`"ChatDotRound"`。

**Q：我改了代碼，但應用沒有更新？**
A：渲染進程（src/）的代碼支持熱更新，保存後自動刷新。但主進程（electron/）代碼改了需要重啟終端。

**Q：`window.electronAPI` 是哪來的？**
A：由 `electron/preload/index.ts` 通過 `contextBridge.exposeInMainWorld('electronAPI', {...})` 注入的。

**Q：TypeScript 報錯 Property 'xxx' does not exist？**
A：可能是 `window.electronAPI` 的類型聲明沒更新。去 `src/types/electron.d.ts` 添加對應類型。

**Q：如何調用後端 API？**
A：使用 `src/api/http-client.ts` 提供的工廠函數：
```typescript
import { createHttpClient } from '@/api/http-client'
const api = createHttpClient(configStore.businessConfig?.ownerSearchApiEndpoint ?? '')
const result = await api.get('/search', { params: { q: 'keyword' } })
```
Token 會被自動注入（auth.interceptor.ts 處理）。

**Q：如何讓窗口最大化按鈕有效果？**
A：TitleBar.vue 已實現，調用 `window.electronAPI.window.maximize()`，主進程接收後執行 `win.maximize()` 或 `win.unmaximize()`。

**Q：浮球怎麼改大小？**
A：修改 `app-config.json` 的 `floatingBall.size`（像素），同時 CSS 的 `width/height` 在 `FloatingBall.vue` 中也要對應修改。
