## 1. 這個項目是什麼？

這是一個**企業桌面客戶端**（產品名 `ichiaDesktop`），用 Electron 打包成 Windows 應用程式（.exe）。
本專案的最終運行環境是 **Windows**，文檔範例與路徑均以 Windows 為主。

主要功能：

| 功能       | 說明                                          |
|----------|---------------------------------------------|
| **統一平台** | 把公司所有內部系統（ERP、BI、OA 等）集中在一個入口，點一下就能打開       |
| **內部功能** | AI 工具（Dify 問答、AiSop 生成）、IT 報修工單等公司內部工具的集中入口 |
| **浮球**   | 最小化後在屏幕邊緣懸浮，隨時快速打開主窗口                       |

## 2. Electron 是什麼？

**Electron = Node.js（後端能力）+ Chromium（網頁渲染）打包成桌面應用**

你用 HTML/CSS/JavaScript 寫界面，Electron 幫你打包成一個可以在 Windows 上雙擊運行的 .exe 文件。

- **普通網頁**：只能在瀏覽器裡跑，不能讀寫本地文件
- **Electron 應用**：既能用網頁技術做界面，又能調用 Node.js 讀寫文件、操作系統托盤等

## 3. 三個進程——最核心的概念

這是理解 Electron 項目最重要的一件事。

```
┌──────────────────────────────────────────────────────────────────┐
│                         你的 Electron 應用                        │
│                                                                  │
│  ┌─────────────────┐      ┌──────────────┐     ┌─────────────┐  │
│  │   主進程 (Main)  │      │  預加載腳本   │     │ 渲染進程    │  │
│  │                 │◄────►│  (Preload)   │◄───►│ (Renderer)  │  │
│  │ electron/main/  │      │electron/     │     │ src/        │  │
│  │ index.ts        │      │preload/      │     │ *.vue       │  │
│  │                 │      │index.ts      │     │             │  │
│  │ Node.js 環境    │      │ 橋樑/安全層  │     │ Chromium    │  │
│  │ 可以讀寫文件    │      │              │     │ 就是網頁    │  │
│  └─────────────────┘      └──────────────┘     └─────────────┘  │
└──────────────────────────────────────────────────────────────────┘
```

### 主進程 (`electron/main/`)

- **是什麼**：整個應用的「大腦」，Node.js 環境
- **能做什麼**：創建窗口、讀寫文件、調用操作系統 API（托盤圖標、通知）
- **不能做什麼**：直接操作網頁 DOM
- **關鍵文件**：`electron/main/index.ts`（入口）

### 渲染進程 (`src/`)

- **是什麼**：顯示界面的「網頁」，Chromium 瀏覽器環境
- **能做什麼**：渲染 HTML/CSS/Vue 組件，響應用戶點擊
- **不能做什麼**：直接讀寫本地文件（安全限制）
- **關鍵文件**：`src/main.ts`（入口）、`src/App.vue`（根組件）

### 預加載腳本 (`electron/preload/`)

- **是什麼**：主進程和渲染進程之間的「安全橋樑」
- **作用**：把主進程的能力「安全地」暴露給網頁，網頁通過 `window.electronAPI.xxx()` 調用
- **為什麼需要它**：直接讓網頁調用 Node.js 太危險（XSS 攻擊），所以要白名單過濾

### 它們怎麼互相通信？

```
渲染進程（Vue 頁面）              主進程（Node.js）
        │                               │
        │  window.electronAPI           │
        │  .window.minimize()           │
        │──────────────────────────────►│  ipcMain.on('window:minimize')
        │                               │  win.minimize()
        │                               │
        │◄──────────────────────────────│  win.webContents.send('push:xxx', data)
        │  window.electronAPI           │
        │  .on('push:xxx', cb)          │
```

## 5. 代碼是怎麼跑起來的？

當你運行 `npm run dev`，發生了以下事情：

**第一步：electron-vite 開始工作**

1. 編譯主進程代碼 `electron/main/index.ts`
2. 編譯預加載腳本 `electron/preload/index.ts`
3. 啟動 Vite 開發服務器（`http://localhost:5173/`），負責實時編譯 Vue 文件
4. 啟動 Electron 應用

**第二步：Electron 主進程啟動（`electron/main/index.ts`）**

按順序依次初始化：
- `ConfigManager` — 讀取 `config/app-config.json`
- `WindowManager` — 創建主窗口（載入 Vite 開發服務器的地址）
- 創建浮球窗口（默認隱藏）
- `registerAllHandlers()` — 注冊所有 IPC 消息監聽器
- `TrayManager` — 創建系統托盤圖標

**第三步：Vue 應用啟動（`src/main.ts`）**

依次執行：
- 創建 Vue 應用，安裝 Pinia（狀態管理）、Router（路由）、ElementPlus（UI 庫）
- 掛載到 `index.html` 的 `<div id="app">`
- `App.vue` 的 `onMounted` 執行：通過 IPC 請求配置文件、嘗試恢復登錄狀態

## 6. 配置文件驅動所有菜單

**這個項目的設計原則：不改代碼，只改配置文件就能改菜單。**

`config/app-config.json` 控制四個地方：

| 配置字段                      | 控制什麼                                      |
|---------------------------|-------------------------------------------|
| `sidebar.items`           | 左側導航菜單有哪些項，順序和圖標是什麼                       |
| `floatingBall.quickMenu`  | 浮球右鍵菜單有哪些項                                |
| `unifiedPlatform.systems` | 統一平台頁顯示哪些系統卡片，以及如何打開它們                    |
| `internalFunctions.tools` | 內部功能頁顯示哪些工具卡片（目前已有 BPM 查詢、AiSop 生成、IT 報修） |

### 配置的流動路徑

```
config/app-config.json
        │  應用啟動時由主進程讀取
        ▼
config-manager.ts（主進程）
        │  渲染進程通過 IPC 請求（window.electronAPI.config.read()）
        ▼
config.store.ts（Pinia Store，存在內存）
        │  Vue 組件從 Store 讀取（響應式，配置變了界面自動更新）
        ▼
SidebarNav.vue      → 讀 configStore.sidebarItems
UnifiedPlatform     → 讀 configStore.platformSystems
InternalFunctions   → 讀 configStore.functionsConfig.tools
```

### 新增一個側邊欄菜單項只需要三步：

1. 在 `app-config.json` 的 `sidebar.items` 加一條
2. 在 `src/router/index.ts` 加對應的路由
3. 在 `src/views/` 新建頁面組件

## 7. Vue 組件：頁面是怎麼搭起來的

### 組件樹（應用打開後的結構）

```
App.vue（根，負責初始化）
  └─ <router-view>（根據 URL 渲染不同東西）
       └─ AppLayout.vue（帶佈局的父路由組件）
            ├─ TitleBar.vue（頂部標題欄，有拖動區）
            └─ <div class="app-body">
                 ├─ SidebarNav.vue（左側導航）
                 │    └─ SidebarNavItem.vue × N
                 └─ <main>（右側內容區）
                      └─ <router-view>（渲染當前頁面）
                           ├─ UnifiedPlatformView.vue   （/unified-platform）
                           ├─ InternalFunctionsView.vue （/internal-functions）
                           ├─ BpmFinderView.vue          （/ai-bpm-finder）
                           ├─ AiSopView.vue              （/ai-sop）
                           └─ ITRepairView.vue           （/it-repair）
```

### 一個 Vue 組件長什麼樣？

每個 `.vue` 文件由三個部分組成：

- **`<script setup lang="ts">`** — TypeScript 邏輯，定義數據、方法、接收父組件傳入的 props
- **`<template>`** — HTML 模板，`{{ }}` 輸出數據，`v-if`/`v-for` 控制渲染，`@click` 綁定事件
- **`<style scoped>`** — CSS 樣式，`scoped` 表示只對本組件生效，不影響其他組件

### `@/` 是什麼意思？

`@/` 是 `src/` 目錄的別名，配置在 `electron.vite.config.ts`。

`import xxx from '@/stores/config.store'` 等同於 `import xxx from 'src/stores/config.store'`。

## 8. Pinia Store：共享數據放哪裡

**問題：A 組件改了數據，B 組件怎麼知道？**

答：把數據放在 Store（全局狀態），所有組件都從同一個地方讀，數據是響應式的。

本項目有三個 Store：

### `config.store.ts` — 應用配置

存放從 `app-config.json` 讀取的配置，任何組件都能訪問：

```typescript
import { useConfigStore } from '@/stores/config.store'
const configStore = useConfigStore()

configStore.sidebarItems        // 側邊欄菜單項（已過濾 enabled: false）
configStore.platformSystems     // 統一平台的系統列表
configStore.functionsConfig     // 內部功能配置（apiBaseUrl/apiTimeout/tools）
configStore.functionsConfig?.tools   // 內部功能工具卡片列表
configStore.floatingBallConfig  // 浮球整體配置
configStore.floatingBallMenuItems    // 浮球右鍵菜單項（已過濾）
configStore.language            // 當前語言設定
```

### `auth.store.ts` — 登錄狀態

存放當前登錄用戶的信息和 Token：

```typescript
import { useAuthStore } from '@/stores/auth.store'
const authStore = useAuthStore()

authStore.accessToken       // 當前 Token（string | null，僅內存）
authStore.isAuthenticated   // 是否已登錄（boolean）
authStore.user              // 用戶信息（UserProfile | null）
authStore.login(user, pwd)  // 登錄
authStore.logout()          // 登出（清除 Token，路由守衛自動跳轉 /login）
```

### `ui.store.ts` — 界面狀態

存放 UI 相關的臨時狀態：

```typescript
import { useUiStore } from '@/stores/ui.store'
const uiStore = useUiStore()

uiStore.sidebarCollapsed     // 側邊欄是否折疊
uiStore.isWindowMaximized    // 窗口是否最大化
uiStore.toggleSidebar()      // 切換側邊欄
```

## 9. IPC 通信：頁面和 Electron 怎麼說話

所有可用方法都在 `window.electronAPI`（由預加載腳本注入），分幾個命名空間：

### 窗口控制

```typescript
window.electronAPI.window.minimize()              // 最小化（同時顯示浮球）
window.electronAPI.window.maximize()             // 最大化/還原
window.electronAPI.window.close()                // 關閉（隱藏到浮球）
window.electronAPI.window.show()                 // 顯示主窗口並聚焦
window.electronAPI.window.hide()                 // 隱藏主窗口並顯示浮球
window.electronAPI.window.isMaximized()          // 查詢是否最大化
window.electronAPI.window.openChild(url, title)  // 打開 Electron 子窗口
```

### 配置

```typescript
const config = await window.electronAPI.config.read()    // 讀取配置
```

> **Token 不走 IPC**：登入後 Token 只存在於渲染進程的 `authStore.accessToken`（內存），
> 不寫磁盤、不經主進程，App 關閉就會清除，下次啟動需要重新登入。

### 監聽主進程推送

主進程可以主動向渲染進程發消息（例如窗口最大化狀態變化）：

```typescript
// 在 onMounted 中監聽
window.electronAPI.on('push:window-maximized', (...args) => {
  uiStore.setWindowMaximized(args[0] as boolean)
})

// 在 onUnmounted 中取消監聽，防止內存洩漏
window.electronAPI.off('push:window-maximized', callback)
```

### 新增 IPC 通信需要改哪些文件？

假設你要添加「獲取版本號」功能，需要依次修改 5 個文件：

1. `electron/shared/ipc-channels.ts` — 添加頻道常量
2. `electron/preload/index.ts` — 暴露方法給渲染進程
3. `electron/main/ipc-handlers/` — 在主進程實現處理邏輯
4. `src/types/electron.d.ts` — 在類型聲明裡添加新方法
5. 在 Vue 組件裡調用 `window.electronAPI.getVersion()`

`app-config.json` 的 `sidebar.items` 中每項有一個 `routeName` 字段，和路由表中的 `name` 字段對應。`SidebarNav.vue` 點擊時調用 `router.push({ name: item.routeName })`。

### 路由守衛做什麼？

1. 如果 `isRestoringSession === true`（App 啟動恢復會話中）→ 直接放行，等恢復完成
2. 如果目標路由需要認證，但用戶未登錄 → 強制跳轉到 `/login`
3. 如果用戶已登錄，卻試圖訪問 `/login` → 直接跳轉到首頁

## 11. 浮球是怎麼實現的

### 為什麼浮球是獨立窗口？

浮球是**第二個 BrowserWindow**，和主窗口完全分離。

| 特性 | 主窗口 | 浮球窗口 |
|------|--------|----------|
| 尺寸 | 1200×800 | 80×80 |
| 透明 | 否 | 是（`transparent: true`） |
| 置頂 | 否 | 是（`alwaysOnTop: true`） |
| 任務欄 | 顯示 | 隱藏（`skipTaskbar: true`） |

圓形效果：窗口本身是矩形的，但透明背景讓四個角不可見，CSS 的 `border-radius: 50%` 讓中間的圓形部分顯示。

### 拖動為什麼不用 `-webkit-app-region: drag`？

因為 `-webkit-app-region: drag` 會**吞掉所有鼠標點擊事件**。

所以我們用了另一種方案：
1. `mousedown` → 告訴主進程「開始拖動」
2. 主進程每 16ms 輪詢一次鼠標位置，更新窗口位置（約 60fps）
3. `mouseup` → 告訴主進程「停止拖動」，觸發邊緣吸附動畫

### 右鍵菜單為什麼用原生菜單？

浮球窗口只有 80×80px，如果在裡面渲染菜單會被窗口邊界裁剪。所以：

- 右鍵點擊浮球 → 通過 IPC 通知主進程 → 主進程用 `Menu.popup()` 彈出原生菜單
- 原生菜單在操作系統層面顯示，不受窗口大小限制

## 12. 我想加一個新頁面，怎麼做？

### 目標：添加「公告板」頁面（三步完成）

**第一步：在 `app-config.json` 加菜單項**

打開 `config/app-config.json`，在 `sidebar.items` 數組末尾加：

```json
{
  "id": "announcement",
  "label": "公告板",
  "icon": "Bell",
  "routeName": "announcement",
  "enabled": true
}
```

圖標名稱去 [Element Plus Icons](https://element-plus.org/zh-CN/component/icon.html) 找，例如 `"Bell"`、`"Grid"`、`"MagicStick"`。

**第二步：創建頁面組件**

新建文件 `src/views/Announcement/AnnouncementView.vue`，最簡單的結構：

```vue
<script setup lang="ts">
// 在這裡寫業務邏輯
</script>

<template>
  <div class="announcement-view">
    <h2>公告板</h2>
    <el-card>公告內容</el-card>
  </div>
</template>

<style scoped>
.announcement-view { padding: 24px; }
</style>
```

**第三步：在路由中注冊**

打開 `src/router/index.ts`，在 `children` 數組加：

```typescript
{
  path: 'announcement',
  name: 'announcement',      // 必須和 app-config.json 的 routeName 一致
  component: () => import('@/views/Announcement/AnnouncementView.vue'),
  meta: { requiresAuth: true, title: '公告板' }
}
```

**完成！** 重啟 `npm run dev`，左側菜單就會出現「公告板」。

## 13. 常見問題 FAQ

**Q：修改了 app-config.json，但界面沒變？**

A：需要重啟應用（`npm run dev`）。配置只在啟動時讀取一次。

**Q：圖標不顯示怎麼辦？**

A：圖標名稱必須和 [Element Plus Icons](https://element-plus.org/zh-CN/component/icon.html) 完全一致，區分大小寫。

**Q：我改了主進程代碼（`electron/` 目錄），但沒有更新？**

A：主進程代碼改了需要重啟終端重新執行 `npm run dev`。渲染進程（`src/`）支持熱更新，保存後自動刷新。

**Q：`window.electronAPI` 是哪來的？**

A：由 `electron/preload/index.ts` 通過 `contextBridge.exposeInMainWorld('electronAPI', {...})` 注入的。

**Q：TypeScript 報錯 `Property 'xxx' does not exist on type 'Window'`？**

A：`window.electronAPI` 的類型聲明沒更新。去 `src/types/electron.d.ts` 添加對應類型。

**Q：如何調用後端 API？**

A：使用 `src/api/http-client.ts` 創建客戶端，Token 會被自動注入：

```typescript
import { createHttpClient } from '@/api/http-client'
const api = createHttpClient('http://your-api.com')
const result = await api.get('/endpoint')
```

**Q：打包後如何確認用的是正式環境地址？**

A：`npm run build:win` 會自動使用 `.env.production` 裡的環境變量。開發時的 `.env.development` 不會進入生產包。

**Q：為什麼開發機是 macOS，但要在 Windows 上跑？**

A：日常開發在 macOS / Windows 都可以（`npm run dev` 都能跑），但**最終交付的是 Windows 安裝包**。所以打包請用
`npm run build:win`，路徑、安裝目錄、系統托盤行為等都以 Windows 為準。
