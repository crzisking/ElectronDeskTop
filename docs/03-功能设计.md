### 3.1 功能概述

當前 AI 快捷功能頁面的實現方式，已從原本規劃的「單頁三個功能面板」調整為「**AI 工具入口卡片頁**」。

實際行為如下：

| 層級 | 當前實現 | 說明 |
|------|----------|------|
| 頁面層 | `AiQuickFunctionsView.vue` | 顯示工具卡片入口，而不是直接顯示三個功能面板 |

### 3.3 流式輸出支持（SSE）

智能問答支持 Server-Sent Events 流式輸出，實現打字機效果：

```typescript
// src/composables/useAiStream.ts
async function sendQuestion(question: string) {
  const response = await fetch(`${config.aiBaseUrl}/qa`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${authStore.accessToken}`,
      'Accept': 'text/event-stream',
    },
    body: JSON.stringify({ question })
  })

  const reader = response.body!.getReader()
  const decoder = new TextDecoder()

  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    const chunk = decoder.decode(value)
    // 解析 SSE data: 字段，追加到 answerText
    parseSSEChunk(chunk).forEach(text => {
      answerText.value += text
    })
  }
}
```

## 4. 業務安排與尋找（原快速聯繫重構）

### 4.1 功能概述

當前第三個主功能頁，已不再是原來的「快速聯繫」，而是重構為「**業務安排與尋找**」。

它包含兩個子功能：

1. **維護業務流水線**
   - 使用 `@vue-flow/core` 提供可視化流程圖編輯能力
   - 支持拖拽節點、連線、節點編輯、保存流程圖數據

2. **尋找業務負責人**
   - 根據業務關鍵詞搜索對應的負責人
   - 展示姓名、部門、職位、職責範圍、郵箱、電話等信息

頁面本身沒有再拆二級路由，而是在 `BusinessView.vue` 中用 `activeView` 做內部切換：

- `home`：功能選擇首頁
- `pipeline`：業務流水線編輯器
- `search`：業務負責人搜索
