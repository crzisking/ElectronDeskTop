# 功能設計文檔

## 1. 登錄功能

### 1.1 設計說明

登錄功能已完整實現，對接公司 Portal OAuth 登錄接口：

- **接口**：`POST {VITE_AUTH_BASE_URL}/api/portal/oauth/login`
- **請求體**：`{ username, password }`
- **回應結構**：`{ code: 200, message: '...', data: { user: UserProfile, token: string } }`

登錄成功後 token 存入 OS Credential Store（通過 `safeStorage + electron-store`），下次啟動時自動恢復會話，不需要重新登錄。

### 1.2 功能清單

| 路由 `/login` | `src/router/index.ts` | ✅ 已實現，路由守衛已啟用 |

### 1.3 路由守衛行為

守衛在 `src/router/index.ts` 的 `beforeEach` 中實現，當前邏輯：

```
每次路由跳轉前：
  1. 若 authStore.isRestoringSession === true → 直接放行（等 restoreSession 完成）
  2. 若目標路由 requiresAuth && !authStore.isAuthenticated → 跳轉 /login
  3. 若已登錄 && 嘗試訪問 /login → 跳轉 /unified-platform
```

### 1.4 開發環境自動登錄

開發時不想每次手動輸入帳號密碼，可在 `.env.development` 設定：

```bash
VITE_DEV_AUTO_LOGIN=true
VITE_DEV_USERNAME=S2403279
VITE_DEV_PASSWORD=yourpassword
```

自動登錄的實現方式（`LoginView.vue` 的 `onMounted`）：

```typescript
onMounted(() => {
  if (import.meta.env.DEV && import.meta.env.VITE_DEV_AUTO_LOGIN === 'true') {
    const username = import.meta.env.VITE_DEV_USERNAME
    const password = import.meta.env.VITE_DEV_PASSWORD
    if (username && password) {
      form.userName = username   // 填入 form（不跳過表單驗證）
      form.password = password
      handleLogin()              // 走正常登錄流程
    }
  }
})
```

> **重要**：自動登錄是「把值填入表單再觸發 handleLogin()」，而不是直接呼叫 authStore.login()。這樣才能走表單驗證，且 UI 上可以看到填入的帳號，方便調試。

### 1.5 auth.store.ts 設計

```typescript
// src/stores/auth.store.ts（Setup Store 寫法）
export const useAuthStore = defineStore('auth', () => {
  // State
  const isAuthenticated = ref<boolean>(false)
  const user = ref<UserProfile | null>(null)
  const accessToken = ref<string | null>(null)   // 僅內存，來源是 OS Credential Store
  const isRestoringSession = ref<boolean>(false)  // 啟動恢復中（路由守衛不攔截）

  // 啟動時恢復會話：讀取 OS Credential Store 中的 token
  async function restoreSession(): Promise<void> { ... }

  // 登錄：呼叫後端 → 存 token → 更新狀態
  async function login(userName: string, password: string): Promise<void> {
    const { token, user: userInfo } = await authApi.login({ username: userName, password })
      as unknown as { token: string; user: UserProfile }
    await window.electronAPI.auth.setToken(token)
    accessToken.value = token
    user.value = userInfo
    isAuthenticated.value = true
  }

  // 登出：清除 token → 重置狀態（路由守衛會自動跳轉 /login）
  async function logout(): Promise<void> { ... }
})
```

### 1.6 注意事項：HTTP 回應資料層提取

`auth.api.ts` 的 `login()` 回傳的**不是**後端 JSON 的原始結構，而是已提取過兩層的 `data` 欄位：

```
後端回傳：{ code:200, message:'...', data:{ user, token } }
  ↓ Axios 攔截器 return response.data
  { code:200, message:'...', data:{ user, token } }
  ↓ auth.api.ts: const { data } = await client.post(...)
  { user, token }   ← authApi.login() 的真實回傳值
```

所以呼叫 `authApi.login()` 後直接解構 `{user, token}` 即可，**不要再讀 `.code` 或 `.data`**。業務碼的錯誤（非 200）已由攔截器統一處理（`ElMessage.error`）。

### 3.3 流式輸出支持（SSE）

智能問答支持 Server-Sent Events 流式輸出，實現打字機效果：

```typescript
// src/composables/useAiStream.ts
async function sendQuestion(question: string) {
  const response = await fetch(`${config.aiBaseUrl}/qa`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${authStore.accessToken}`,
      'Accept': 'text/event-stream',
    },
    body: JSON.stringify({ question })
  })

  const reader = response.body!.getReader()
  const decoder = new TextDecoder()

  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    const chunk = decoder.decode(value)
    // 解析 SSE data: 字段，追加到 answerText
    parseSSEChunk(chunk).forEach(text => {
      answerText.value += text
    })
  }
}
```
