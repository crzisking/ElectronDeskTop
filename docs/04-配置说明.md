# 配置說明文檔

## 1. 概述

`app-config.json` 是應用的**唯一外部配置文件**。修改這個文件可以在不重新編譯的情況下調整所有菜單、系統鏈接、功能開關和 API 地址。

**文件位置：**

| 環境           | 路徑                                                             |
|--------------|----------------------------------------------------------------|
| 開發環境         | `config/app-config.json`（項目根目錄）                                |
| Windows 生產環境 | `C:\Users\<User>\AppData\Roaming\ichiaDesktop\app-config.json` |

> 生產環境的 `userData` 路徑由 Electron 根據 `package.json` 的 `productName`（`ichiaDesktop`）決定。
> 安裝包啟動時會將 `extraResources` 中的默認 `app-config.json` 複製到此目錄，每次升級重新覆蓋以保持與代碼同步。

**載入時機：** 應用啟動時由主進程讀取，通過 IPC 發送至渲染進程，存儲在 `configStore`。

---

## 2. 頂層結構

配置文件有 7 個頂層字段：

| 字段                  | 說明                                                        |
|---------------------|-----------------------------------------------------------|
| `version`           | 配置文件版本號，格式 `"1.0.0"`                                      |
| `app`               | 全局設置（語言、是否開機自啟、是否靜默啟動）                                    |
| `sidebar`           | 左側導航欄菜單配置                                                 |
| `floatingBall`      | 浮球外觀和快捷菜單                                                 |
| `unifiedPlatform`   | 統一平台頁的系統卡片列表                                              |
| `internalFunctions` | 內部功能頁的工具卡片列表（含 AI 工具入口）                                   |
| `update`            | 自動更新配置（feedUrl、每日檢查時刻等，詳見 [06-自动更新设计.md](./06-自动更新设计.md)） |

---

## 3. 各字段詳解

### 3.1 `app` — 全局設置

| 字段 | 類型 | 說明 |
|------|------|------|
| `language` | `"zh-TW"` \| `"en"` | 界面語言 |
| `startMinimized` | boolean | `true` 時啟動不顯示主窗口，直接顯示浮球 |
| `launchOnStartup` | boolean | 是否加入系統開機自啟動 |

> 應用只支持單一亮色主題，不提供 `theme` 切換字段。

---

### 3.2 `sidebar.items` — 左側導航菜單

數組，每一項是一個菜單條目，**順序即為顯示順序**。

| 字段 | 類型 | 必填 | 說明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一標識，建議與 `routeName` 相同 |
| `label` | string | ✅ | 顯示文字 |
| `icon` | string | ✅ | Element Plus 圖標名，如 `"Grid"`, `"MagicStick"`, `"Cpu"` |
| `routeName` | string | ✅ | Vue Router 路由 name，必須和 `router/index.ts` 中的 `name` 字段對應 |
| `enabled` | boolean | ✅ | `false` 時該菜單項不渲染（隱藏但不刪除） |
| `badge` | string | ❌ | 菜單項右側徽標，如 `"New"`, `"99+"` |

圖標名稱查詢：[Element Plus Icons](https://element-plus.org/zh-CN/component/icon.html)

---

### 3.3 `floatingBall` — 浮球配置

**外觀設置：**

| 字段                | 類型         | 說明                   |
|-------------------|------------|----------------------|
| `size`            | number     | 浮球直徑（像素），默認 80       |
| `opacity`         | number     | 透明度 0.0 ~ 1.0，默認 0.9 |
| `defaultPosition` | `{ x, y }` | 初始屏幕坐標               |
| `snapToEdge`      | boolean    | 停止拖動後是否吸附到最近屏幕邊緣     |

**快捷菜單 `quickMenu`：**

| 字段 | 類型 | 說明 |
|------|------|------|
| `id` | string | 唯一標識 |
| `label` | string | 菜單文字 |
| `icon` | string | Element Plus 圖標名（可選） |
| `action` | object | 點擊後執行的操作，見下方說明 |
| `enabled` | boolean | `false` 時不顯示 |
| `separator` | boolean | `true` 時在此項後渲染分隔線 |

**`action.type` 取值：**

| `type` | 額外字段 | 行為 |
|--------|----------|------|
| `show-main-window` | 無 | 顯示主窗口，隱藏浮球 |
| `navigate` | `routeName: string` | 顯示主窗口並導航到指定路由 |
| `open-url` | `url: string`, `target: "browser"\|"iframe"` | 在瀏覽器或 iframe 中打開 URL |
| `quit-app` | 無 | `app.quit()` 完全退出 |

---

### 3.4 `unifiedPlatform.systems` — 系統鏈接列表

每一項代表統一平台頁的一個系統卡片：

| 字段 | 類型 | 必填 | 說明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一標識 |
| `name` | string | ✅ | 系統名稱 |
| `description` | string | ✅ | 系統描述（顯示在卡片上） |
| `url` | string | ✅ | 系統訪問地址 |
| `iconUrl` | string | ❌ | 系統圖標 URL，無則顯示名稱首字母 |
| `openMode` | enum | ✅ | `"iframe"` / `"electron-window"` / `"external-browser"` |
| `ssoEnabled` | boolean | ✅ | `true` 時自動將 Auth Token 注入 URL |
| `ssoTokenParam` | string | 條件 | Token 注入的 URL 參數名，`ssoEnabled: true` 時必填 |

---

### 3.5 `internalFunctions` — 內部功能工具卡片

這裡配置的是「內部功能」頁面（路由 `/internal-functions`）顯示的工具卡片列表。

**基礎設置：**

| 字段 | 類型 | 說明 |
|------|------|------|
| `apiBaseUrl` | string | AI 後端 API 基礎地址（預留，當前 AI 功能走 Dify SSE） |
| `apiTimeout` | number | AI API 超時時間（毫秒） |

**`tools` 數組（每一項是一個工具卡片）：**

| 字段 | 類型 | 必填 | 說明 |
|------|------|------|------|
| `id` | string | ✅ | 唯一標識 |
| `name` | string | ✅ | 工具名稱 |
| `description` | string | ✅ | 工具功能描述 |
| `icon` | string | ✅ | Element Plus 圖標名 |
| `enabled` | boolean | ✅ | `false` 時卡片不顯示 |
| `openMode` | `"page"` \| `"external"` | ✅ | `"page"` 跳轉子路由；`"external"` 在瀏覽器打開 |
| `routeName` | string | 條件 | `openMode: "page"` 時必填，對應路由 name |
| `url` | string | 條件 | `openMode: "external"` 時必填 |

---

### 3.6 `update` — 自動更新

| 字段                     | 類型                                  | 說明                                                 |
|------------------------|-------------------------------------|----------------------------------------------------|
| `enabled`              | boolean                             | 自動更新總開關，`false` 時 UpdateManager 不啟動                |
| `feedUrl`              | string                              | 內網更新源（結尾必須帶 `/`），需放有 `latest.yml` + 安裝包 + blockmap |
| `channel`              | `"latest"` \| `"beta"` \| `"alpha"` | 對應服務器上的 `latest.yml` / `beta.yml` / `alpha.yml`    |
| `dailyCheckTime`       | string                              | 每日定時檢查時刻（24 小時制 `HH:MM`），空字串 `""` 表示停用定時           |
| `autoDownload`         | boolean                             | 發現新版時是否自動背景下載                                      |
| `autoInstallOnAppQuit` | boolean                             | 退出時是否靜默安裝（建議 `false`，由用戶按「立即重啟」）                   |

完整流程、IPC 頻道、發布步驟見 [06-自动更新设计.md](./06-自动更新设计.md)。

## 5. 配置載入流程

```
應用啟動
    │
    ▼
主進程讀取 app-config.json
    ├─ 文件存在 → 解析 JSON → 與默認值 deepMerge
    └─ 文件不存在 → 從安裝包複製默認配置到 userData → 讀取
    │
    ▼
渲染進程通過 IPC 請求（CONFIG_READ）
    │
    ▼
configStore.loadConfig()
    ├─ 存入 configStore.appConfig
    └─ 各組件通過 getter 響應式讀取（自動重新渲染）
```

---

## 6. 配置熱更新（可選）

如需管理員更新配置後無需重啟應用立即生效：

1. 主進程用 `fs.watchFile()` 監聽 `app-config.json` 變更
2. 文件變更後重新讀取並驗證
3. 通過 `mainWindow.webContents.send(IpcChannels.PUSH_CONFIG_CHANGED, newConfig)` 推送渲染進程
4. 渲染進程監聽並更新 `configStore`，組件自動重新渲染

> **注意**：浮球外觀變化（`size`/`opacity`）不走熱更新，需重啟後生效（浮球窗口已按舊尺寸創建）。
