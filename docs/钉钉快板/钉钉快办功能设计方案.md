# 钉钉快办功能设计方案

> 桌面端集成钉钉快办的任务管理：列协作空间、列/查/增/改/删任务。
> **不做提醒**（钉钉 App 自带）、**不做本地缓存**（数据全部走后端实时）。
>
> 技术栈：Vue 3 + TypeScript + Pinia + Element Plus；后端 .NET（tmbom），已有钉钉授权代码。

---

## 一、范围与非范围

### 做什么

- 列出当前用户的协作空间 / 项目
- 列出任务（按空间筛选 / 按完成态筛选 / 按截止日期筛选）
- 查任务详情
- 创建任务（项目任务 + 自由任务）
- 编辑任务、改完成态、删除任务

### 不做什么（明确划掉，避免后面又被加回来）

- ❌ **定时提醒**：钉钉 App 已有同样能力，桌面端做了反而扰民
- ❌ **本地 SQLite 缓存**：钉钉数据由后端实时代理，前端只读 HTTP；离线不在 P0 范围
- ❌ **AI 摘要 / 工作统计**：等任务 CRUD 跑稳了再说

```
│  │    api.ts     ─ 走 createHttpClient,带 JWT 拦截器        │  │

│  └──────────────────────────────────────────────────────────┘  │
└──────────────────────────┬─────────────────────────────────────┘
                           │ HTTPS, Bearer JWT (ichia 自家)
                           ▼
┌────────────────────────────────────────────────────────────────┐
│         tmbom .NET Backend  (controllers/DingtalkController)   │
│   /api/dingtalk/projects   /tasks   /users (后期)              │
│                                                                │
│   ┌──────────────────────────────────────────────────────┐     │
│   │  既有钉钉授权代码:accessToken 获取/刷新/调用钉钉      │     │
│   │  ↑ 前端不感知,后端代理所有钉钉 API                    │     │
│   └──────────────────────────────────────────────────────┘     │
└──────────────────────────┬─────────────────────────────────────┘
                           │
                           ▼
                   钉钉开放平台 OpenAPI
```

**关键决策**：

- 后端是钉钉的**唯一代理**，前端永远不直接打钉钉 API，授权 token 不出后端
- 主进程**不参与**钉钉模块，HTTP 全在 renderer 走 `createHttpClient`（跟 work-collect 一致）
- 没有 SQLite，没有 IPC（除了导航这种通用功能）

## 四、后端 API 契约

> 前端面对的接口，**不是**钉钉原 API。tmbom 后端自己规整字段、隐藏 token、做错误码标准化。
> 全部走 ichia 既有响应包装 `{ code, message, data }`，前端拦截器剥外层只拿 `data`。

### 4.1 协作空间

| Method | Path                                 | 说明          |
|--------|--------------------------------------|-------------|
| GET    | `/api/dingtalk/projects`             | 当前用户参与的协作空间 |
| POST   | `/api/dingtalk/projects`             | 创建协作空间      |
| PUT    | `/api/dingtalk/projects/{projectId}` | 更新协作空间      |
| DELETE | `/api/dingtalk/projects/{projectId}` | 删除协作空间      |

**`GET /projects`** — query: `maxResults` (默认 50，上限 300) / `nextToken` / `name`（模糊匹配）

```json
// data
{
  "projects": [
    {
      "id": "xxx",
      "name": "前端组 2026Q1",
      "description": "",
      "logo": "https://...",
      "creatorId": "userA",
      "createdAt": 1735660800000,
      "updatedAt": 1735660800000
    }
  ],
  "nextToken": "abc"
  // 没下一页时为 null
}
```

> 时间字段全部用 **Unix ms（number）**，避免 ISO 字符串跨时区错一天。后端转换好再吐。

**`POST /projects`** — body `{ name, description? }`，返回新建的 project 对象

### 4.2 任务

| Method | Path                                  | 说明         |
|--------|---------------------------------------|------------|
| GET    | `/api/dingtalk/tasks`                 | 任务列表（分页）   |
| GET    | `/api/dingtalk/tasks/{taskId}`        | 任务详情       |
| POST   | `/api/dingtalk/tasks`                 | 创建任务       |
| PUT    | `/api/dingtalk/tasks/{taskId}`        | 更新任务（部分字段） |
| PATCH  | `/api/dingtalk/tasks/{taskId}/status` | 改完成态       |
| DELETE | `/api/dingtalk/tasks/{taskId}`        | 删除任务       |

**`GET /tasks`** — query：

| 字段                            | 类型       | 说明                    |
|-------------------------------|----------|-----------------------|
| `projectId`                   | string?  | 不传 = 当前用户的全部任务（含自由任务） |
| `isDone`                      | boolean? | 不传 = 不过滤              |
| `dueDateStart` / `dueDateEnd` | number?  | Unix ms，闭区间           |
| `maxResults`                  | number?  | 默认 50，上限 200          |
| `nextToken`                   | string?  | 分页 token              |

```json
// data
{
  "tasks": [
    /* DingtalkTask */
  ],
  "nextToken": null
}
```

**`POST /tasks`** — body：

```json
{
  "type": "project | free",
  "projectId": "xxx",
  // type=project 时必填
  "content": "完成季度报表",
  // 任务标题
  "executorId": "userA",
  // 可空(自己)
  "dueDate": 1735660800000,
  // 可空
  "note": "需要包含 Q4 数据",
  "priority": 0,
  // 0 无 / 1 低 / 2 中 / 3 高
  "involveMembers": [
    "userB"
  ],
  // 可空
  "visible": "involves | members"
}
```

返回完整任务对象。

**`PATCH /tasks/{taskId}/status`** — body `{ isDone: true }`，返回 204 / 空。

### 4.3 用户/成员（后期实现，前端先占位）

> 创建任务需要 `executorId` / `involveMembers`，前端必须能选人。
> **此 API 后端尚未实现**，前端在 `MemberPicker` 内部按以下契约预留，到时直接对接。

| Method | Path                                         | 说明              |
|--------|----------------------------------------------|-----------------|
| GET    | `/api/dingtalk/projects/{projectId}/members` | 某空间的成员列表（项目任务用） |
| GET    | `/api/dingtalk/users/search?keyword=xxx`     | 全员搜索（自由任务用）     |

```json
// data
{
  "members": [
    {
      "userId": "xxx",
      "name": "张三",
      "avatar": "https://...",
      "title": "工程师"
    }
  ]
}
```

**MVP 期间**：`TaskForm` 的执行者/参与者字段先用 **文本输入 userId** 顶着，能跑就行；成员 API 上线后切到 `MemberPicker`
即可，不动 store / api。

## 五、前端 API 模块

```ts
// src/features/dingtalk/api.ts
import {createHttpClient} from '@/api/http-client'
import type {DingtalkProject, DingtalkTask, CreateTaskRequest, PageResult} from './types'

// 跟 work-collect 同模式:独立 env var,默认 fallback 全 URL(renderer 没有 same-origin)
const BASE_URL: string =
    (import.meta.env.VITE_DINGTALK_API_URL as string | undefined) ?? 'http://localhost:5247'

let _client: ReturnType<typeof createHttpClient> | null = null
function getClient() {
    if (!_client) _client = createHttpClient(BASE_URL, 30_000)
  return _client
}

export const dingtalkApi = {
    // ── 协作空间 ──────────────────────────────────────
    listProjects: (params?: { maxResults?: number; nextToken?: string; name?: string }) =>
        getClient().get<PageResult<DingtalkProject, 'projects'>>('/api/dingtalk/projects', {params}),

    createProject: (data: { name: string; description?: string }) =>
        getClient().post<DingtalkProject>('/api/dingtalk/projects', data),

    updateProject: (id: string, data: { name?: string; description?: string }) =>
        getClient().put<void>(`/api/dingtalk/projects/${id}`, data),

    deleteProject: (id: string) =>
        getClient().delete<void>(`/api/dingtalk/projects/${id}`),

    // ── 任务 ──────────────────────────────────────────
    listTasks: (params?: {
    projectId?: string
    isDone?: boolean
        dueDateStart?: number
        dueDateEnd?: number
        maxResults?: number
        nextToken?: string
    }) => getClient().get<PageResult<DingtalkTask, 'tasks'>>('/api/dingtalk/tasks', {params}),

    getTask: (id: string) =>
        getClient().get<DingtalkTask>(`/api/dingtalk/tasks/${id}`),

  createTask: (data: CreateTaskRequest) =>
      getClient().post<DingtalkTask>('/api/dingtalk/tasks', data),

    updateTask: (id: string, data: Partial<CreateTaskRequest>) =>
        getClient().put<void>(`/api/dingtalk/tasks/${id}`, data),

    setTaskDone: (id: string, isDone: boolean) =>
        getClient().patch<void>(`/api/dingtalk/tasks/${id}/status`, {isDone}),

    deleteTask: (id: string) =>
        getClient().delete<void>(`/api/dingtalk/tasks/${id}`),
}
```

## 六、Pinia Store

```ts
// src/features/dingtalk/store.ts
export const useDingtalkStore = defineStore('dingtalk', () => {
    // ── State ─────────────────────────────────────────
  const projects = ref<DingtalkProject[]>([])
  const tasks = ref<DingtalkTask[]>([])
    const tasksNextToken = ref<string | null>(null)
  const currentTask = ref<DingtalkTask | null>(null)

    const loading = reactive({projects: false, tasks: false, task: false})

    /** 当前任务列表的查询条件,刷新时复用 */
    const taskQuery = ref<Parameters<typeof dingtalkApi.listTasks>[0]>({})

    // ── Getters ───────────────────────────────────────
    /** 今日待办(due 在今天 00:00 ~ 23:59 之间,未完成) */
  const todayTasks = computed(() => {
        const {start, end} = todayRange()
        return tasks.value.filter(
            (t) => !t.isDone && t.dueDate != null && t.dueDate >= start && t.dueDate <= end
        )
  })

    /** 逾期(due < 今天 00:00,未完成) —— 「今天到期」不算逾期 */
  const overdueTasks = computed(() => {
        const {start} = todayRange()
        return tasks.value.filter((t) => !t.isDone && t.dueDate != null && t.dueDate < start)
  })

    /** 今日完成率(分母用所有 due=今天 的任务,含已完成) */
  const todayCompletionRate = computed(() => {
        const {start, end} = todayRange()
        const todayAll = tasks.value.filter(
            (t) => t.dueDate != null && t.dueDate >= start && t.dueDate <= end
        )
        if (!todayAll.length) return 0
        const done = todayAll.filter((t) => t.isDone).length
        return Math.round((done / todayAll.length) * 100)
  })

    // ── Actions ───────────────────────────────────────
    async function fetchProjects(params?: Parameters<typeof dingtalkApi.listProjects>[0]) {
        loading.projects = true
    try {
        const res = await dingtalkApi.listProjects(params)
      projects.value = res.projects
    } finally {
        loading.projects = false
    }
  }

    async function fetchTasks(params?: typeof taskQuery.value) {
        loading.tasks = true
    try {
        taskQuery.value = params ?? {}
        const res = await dingtalkApi.listTasks(params)
      tasks.value = res.tasks
        tasksNextToken.value = res.nextToken
    } finally {
        loading.tasks = false
    }
  }

    /** 加载下一页,append 到 tasks 后面 */
    async function loadMoreTasks() {
        if (!tasksNextToken.value) return
        const res = await dingtalkApi.listTasks({...taskQuery.value, nextToken: tasksNextToken.value})
        tasks.value.push(...res.tasks)
        tasksNextToken.value = res.nextToken
  }

    async function fetchTask(taskId: string) {
        loading.task = true
    try {
        currentTask.value = await dingtalkApi.getTask(taskId)
    } finally {
        loading.task = false
    }
    }

    async function createTask(data: CreateTaskRequest) {
        const created = await dingtalkApi.createTask(data)
        // 创建后乐观插入,不必整页 refetch(钉钉 API 有限流)
        tasks.value.unshift(created)
        return created
    }

    async function updateTask(taskId: string, data: Partial<CreateTaskRequest>) {
        await dingtalkApi.updateTask(taskId, data)
        const idx = tasks.value.findIndex((t) => t.id === taskId)
        if (idx >= 0) tasks.value[idx] = {...tasks.value[idx], ...data} as DingtalkTask
        if (currentTask.value?.id === taskId) Object.assign(currentTask.value, data)
  }

    async function toggleDone(taskId: string, isDone: boolean) {
        await dingtalkApi.setTaskDone(taskId, isDone)
        const t = tasks.value.find((x) => x.id === taskId)
        if (t) t.isDone = isDone
        if (currentTask.value?.id === taskId) currentTask.value.isDone = isDone
  }

    async function deleteTask(taskId: string) {
        await dingtalkApi.deleteTask(taskId)
        tasks.value = tasks.value.filter((t) => t.id !== taskId)
        if (currentTask.value?.id === taskId) currentTask.value = null
  }

  return {
      projects, tasks, tasksNextToken, currentTask, loading,
      todayTasks, overdueTasks, todayCompletionRate,
      fetchProjects, fetchTasks, loadMoreTasks, fetchTask,
      createTask, updateTask, toggleDone, deleteTask,
  }
})

function todayRange() {
    const s = new Date();
    s.setHours(0, 0, 0, 0)
    const e = new Date();
    e.setHours(23, 59, 59, 999)
    return {start: s.getTime(), end: e.getTime()}
}
```

**几个关键点**：

- `tasksNextToken` 暴露给 UI，是否显示「加载更多」按钮看它是不是 null
- 创建/更新/删除全部走**乐观更新**，不整列 refetch；钉钉 API 有限流，省一次是一次
- 时间字段全是 number（Unix ms），computed 内不再 `new Date(string)`，免去时区坑

## 七、类型定义

```ts
// src/features/dingtalk/types.ts
export interface DingtalkProject {
  id: string
  name: string
  description?: string
  logo?: string
  creatorId: string
    createdAt: number   // Unix ms
    updatedAt: number
}

export interface DingtalkTask {
  id: string
    projectId?: string         // 自由任务无值
  content: string            // 任务标题
    note?: string              // 备注
    executorId?: string
    executorName?: string
    creatorId: string
    isDone: boolean
    dueDate?: number           // Unix ms
    priority: 0 | 1 | 2 | 3    // 0 无 / 1 低 / 2 中 / 3 高
    involveMemberIds?: string[]
    createdAt: number
    updatedAt: number
    accomplishedAt?: number
}

export interface CreateTaskRequest {
    type: 'project' | 'free'
    projectId?: string
    content: string
    executorId?: string
    dueDate?: number
    note?: string
    priority?: 0 | 1 | 2 | 3
    involveMembers?: string[]
    visible?: 'involves' | 'members'
}

/** 钉钉的 cursor 分页:用泛型让 nextToken 跟 list key 解耦 */
export type PageResult<T, K extends string> = {
    [P in K]: T[]
} & {
    nextToken: string | null
}
// 用法:PageResult<DingtalkTask, 'tasks'> → { tasks: DingtalkTask[]; nextToken }
```

## 八、UI 注意点

### 8.1 TaskList

- 顶部筛选条：协作空间下拉 / 完成态 toggle / 日期范围
- 行内操作：勾选框（完成态）/ 优先级 tag / 详情按钮 / 删除按钮
- 底部「加载更多」按钮：`tasksNextToken !== null` 时显示
- **逾期标红**：行有 `overdueTasks` 包含则左边一条红线 + 截止日期文字 `--el-color-danger`

### 8.2 TaskForm（对话框）

- `type=project` 时显示协作空间下拉、`type=free` 时隐藏
- `executorId` / `involveMembers`：MVP 用 `el-input` 输 userId，**留 TODO 注释**等 `MemberPicker` 上线切换
- `priority` 用 `el-radio-group` 0/1/2/3 + 中文标签
- `dueDate` 用 `el-date-picker type="datetime"`，提交时 `.getTime()` 转 number

### 8.3 i18n

- 对齐 work-collect，所有文案进 `src/locales/zh-TW.json` + `en.json`，根 key `dingtalk.*`
- 类别 / 优先级 / 错误信息全部走 i18n，不写死中文

### 8.4 错误处理

- 后端用 ichia 既有 `{ code, message }` 包装，HTTP client 已统一抛 `ApiError`
- TaskForm 内 try/catch → `ElMessage.error(err.message ?? '操作失败')`
- 钉钉限流（后端透传 429 或自定 code）走同样路径，不必特别处理

## 十、显式不做（保留给未来讨论）

## 十一、约定与备注

1. **时间统一 Unix ms（number）**：后端在出口转好，前端不解析 ISO 字符串
2. **环境变量**：`VITE_DINGTALK_API_URL` 指向 tmbom 后端根地址，path 写在每个 API 调用里
3. **认证**：复用 ichia JWT，钉钉 accessToken 全在 tmbom 后端，前端**不感知**
4. **响应包装**：所有接口走 `{ code, message, data }`，HTTP 客户端拦截器会剥到 `data`
5. **目录名 typo**：`docs/钉钉快板/` 应改为 `docs/钉钉快办/`（板 → 办），下个 PR 一并修
