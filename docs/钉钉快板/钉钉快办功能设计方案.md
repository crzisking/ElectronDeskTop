# 钉钉快办功能设计方案

> 本文档设计基于钉钉快办 API 的前端功能，目标是在桌面端实现任务增删改查，并提供 AI 智能提醒功能。
>
> **技术栈**: Vue 3 + TypeScript + Pinia + Element Plus + .NET Backend

## 三、后端 API 对接设计

> 以下为前端需要调用的 .NET Backend API 接口设计。
> **后端接口由 .NET 端实现，前端仅负责调用。**

### 3.1 协作空间

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/dingtalk/projects` | 获取协作空间列表 |
| POST | `/api/dingtalk/projects` | 创建协作空间 |
| PUT | `/api/dingtalk/projects/{projectId}` | 更新协作空间 |
| DELETE | `/api/dingtalk/projects/{projectId}` | 删除协作空间 |

**GET /api/dingtalk/projects**

Query 参数:
- `maxResults`: number (默认10，最大300)
- `nextToken`: string (分页标识)
- `name`: string (模糊搜索名称)

响应:
```json
{
  "data": {
    "projects": [
      {
        "id": "string",
        "name": "string",
        "description": "string",
        "logo": "string",
        "creatorId": "string",
        "createdAt": "2026-01-01T00:00:00Z",
        "updatedAt": "2026-01-01T00:00:00Z"
      }
    ],
    "nextToken": "string"
  }
}
```

**POST /api/dingtalk/projects**

Body:
```json
{
  "name": "string",
  "description": "string"
}
```

### 3.2 任务管理

| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/dingtalk/tasks` | 获取任务列表 |

| PATCH | `/api/dingtalk/tasks/{taskId}/status` | 更新任务状态 |
| DELETE | `/api/dingtalk/tasks/{taskId}` | 删除任务 |

**GET /api/dingtalk/tasks**

Query 参数:
- `projectId`: string (协作空间ID，不传则查全部)
- `isDone`: boolean (筛选完成状态)
- `dueDateStart`: string (截止日期范围开始，ISO8601)

```json
{
  "type": "project" | "free",

  "content": "string",
  "executorId": "string",
  "dueDate": "2026-01-01T00:00:00Z",
  "note": "string",
  "priority": 0,
  "involveMembers": ["userId1", "userId2"],
  "visible": "involves" | "members"
```

**PATCH /api/dingtalk/tasks/{taskId}/status**

## 四、前端 API 模块设计

**文件位置**: `src/features/dingtalk/api.ts`

```typescript
/**
 * 钉钉快办 API 模块
 *
 * 调用 .NET Backend API: /api/dingtalk/*
 * 使用 createHttpClient 工厂，复用 auth 拦截器。
 */

import {createHttpClient} from '@/api/http-client'
import type {
  DingtalkProject,
  DingtalkTask,
  ReminderSettings,
  ReminderHistory,
  SyncStatus,
} from './types'

// API Base URL - 从环境变量或配置读取
const BASE_URL = import.meta.env.VITE_DINGTALK_API_URL ?? '/api/dingtalk'

let _client: ReturnType<typeof createHttpClient> | null = null
function getClient() {
  if (!_client) {
    _client = createHttpClient(BASE_URL, 30000)
  }
  return _client
}

// ── API 函数 ────────────────────────────────────────────────

/** 协作空间 */
export const dingtalkApi = {
  getProjects: (params?: { maxResults?: number; nextToken?: string; name?: string }) =>
    getClient().get<{ projects: DingtalkProject[]; nextToken?: string }>('/projects', { params }),

  createProject: (data: { name: string; description?: string }) =>
    getClient().post<DingtalkProject>('/projects', data),

  updateProject: (projectId: string, data: { name?: string; description?: string }) =>
    getClient().put<void>(`/projects/${projectId}`, data),

  deleteProject: (projectId: string) =>
    getClient().delete<void>(`/projects/${projectId}`),

  /** 任务 */
  getTasks: (params?: {
    projectId?: string
    isDone?: boolean
    dueDateStart?: string
    dueDateEnd?: string
  }) => getClient().get<{ tasks: DingtalkTask[] }>('/tasks', { params }),

  getTask: (taskId: string) =>
    getClient().get<DingtalkTask>(`/tasks/${taskId}`),

  createTask: (data: CreateTaskRequest) =>
    getClient().post<DingtalkTask>('/tasks', data),

  updateTask: (taskId: string, data: Partial<CreateTaskRequest>) =>
    getClient().put<void>(`/tasks/${taskId}`, data),

  updateTaskStatus: (taskId: string, isDone: boolean) =>
    getClient().patch<void>(`/tasks/${taskId}/status`, { isDone }),

  deleteTask: (taskId: string) =>
    getClient().delete<void>(`/tasks/${taskId}`),

  /** 同步 */
  fullSync: () => getClient().post<SyncStatus>('/sync/full'),
  incrementalSync: () => getClient().post<SyncStatus>('/sync/incremental'),
  getSyncStatus: () => getClient().get<SyncStatus>('/sync/status'),

  /** 提醒 */
  getReminderSettings: () =>
    getClient().get<ReminderSettings>('/reminder/settings'),

  updateReminderSettings: (settings: ReminderSettings) =>
    getClient().put<void>('/reminder/settings', settings),

  getReminderHistory: (params?: { startDate?: string; endDate?: string; limit?: number }) =>
    getClient().get<{ history: ReminderHistory[] }>('/reminder/history', { params }),

  triggerMorningReminder: () => getClient().post<void>('/reminder/trigger/morning'),
  triggerEveningReminder: () => getClient().post<void>('/reminder/trigger/evening'),
}

// ── 类型定义 ────────────────────────────────────────────────

export interface CreateTaskRequest {
  type: 'project' | 'free'
  projectId?: string
  content: string
  executorId?: string
  dueDate?: string
  note?: string
  priority?: number
  involveMembers?: string[]
  visible?: 'involves' | 'members'
}
```

## 五、前端 Store 设计

**文件位置**: `src/features/dingtalk/store.ts`

```typescript
/**
 * 钉钉快办 Pinia Store
 *
 * 职责:
 *  - 管理协作空间、任务列表状态
 *  - 管理提醒设置
 *  - 提供 computed 属性 (今日任务、逾期任务、完成率等)
 */

import {defineStore} from 'pinia'
import {ref, computed} from 'vue'
import {dingtalkApi} from './api'
import type {DingtalkProject, DingtalkTask, ReminderSettings} from './types'

export const useDingtalkStore = defineStore('dingtalk', () => {
  // ── State ────────────────────────────────────────────────
  const projects = ref<DingtalkProject[]>([])
  const currentProject = ref<DingtalkProject | null>(null)
  const tasks = ref<DingtalkTask[]>([])
  const currentTask = ref<DingtalkTask | null>(null)
  const syncStatus = ref<'idle' | 'syncing' | 'error'>('idle')
  const lastSyncTime = ref<number | null>(null)
  const reminderSettings = ref<ReminderSettings>({
    morningReminderTime: '08:00',
    eveningReminderTime: '17:00',
    morningEnabled: true,
    eveningEnabled: true,
    weekendEnabled: false,
  })
  const loading = ref({projects: false, tasks: false, sync: false})

  // ── Getters ──────────────────────────────────────────────
  /** 今日待办任务 */
  const todayTasks = computed(() => {
    const today = new Date()
    today.setHours(0, 0, 0, 0)
    const tomorrow = new Date(today)
    tomorrow.setDate(tomorrow.getDate() + 1)

    return tasks.value.filter(t => {
      if (!t.dueDate) return false
      const due = new Date(t.dueDate)
      return due >= today && due < tomorrow
    })
  })

  /** 明日待办任务 */
  const tomorrowTasks = computed(() => {
    const tomorrow = new Date()
    tomorrow.setDate(tomorrow.getDate() + 1)
    tomorrow.setHours(0, 0, 0, 0)
    const dayAfter = new Date(tomorrow)
    dayAfter.setDate(dayAfter.getDate() + 1)

    return tasks.value.filter(t => {
      if (!t.dueDate) return false
      const due = new Date(t.dueDate)
      return due >= tomorrow && due < dayAfter
    })
  })

  /** 逾期未完成任务 */
  const overdueTasks = computed(() => {
    const now = new Date()
    now.setHours(0, 0, 0, 0)
    return tasks.value.filter(t => {
      if (!t.dueDate || t.isDone) return false
      return new Date(t.dueDate) < now
    })
  })

  /** 今日完成率 */
  const todayCompletionRate = computed(() => {
    const today = todayTasks.value
    if (today.length === 0) return 0
    const completed = today.filter(t => t.isDone).length
    return Math.round((completed / today.length) * 100)
  })

  // ── Actions ──────────────────────────────────────────────
  /** 获取协作空间列表 */
  async function fetchProjects(params?: { maxResults?: number; nextToken?: string; name?: string }) {
    loading.value.projects = true
    try {
      const res = await dingtalkApi.getProjects(params)
      projects.value = res.projects
    } finally {
      loading.value.projects = false
    }
  }

  /** 创建协作空间 */
  async function createProject(data: { name: string; description?: string }) {
    const res = await dingtalkApi.createProject(data)
    await fetchProjects()
    return res
  }

  /** 获取任务列表 */
  async function fetchTasks(params?: {
    projectId?: string
    isDone?: boolean
    dueDateStart?: string
    dueDateEnd?: string
  }) {
    loading.value.tasks = true
    try {
      const res = await dingtalkApi.getTasks(params)
      tasks.value = res.tasks
    } finally {
      loading.value.tasks = false
    }
  }

  /** 获取任务详情 */
  async function fetchTaskDetail(taskId: string) {
    const res = await dingtalkApi.getTask(taskId)
    currentTask.value = res
  }

  /** 创建任务 */
  async function createTask(data: Parameters<typeof dingtalkApi.createTask>[0]) {
    const res = await dingtalkApi.createTask(data)
    await fetchTasks()
    return res
  }

  /** 切换任务完成状态 */
  async function toggleTaskStatus(taskId: string, isDone: boolean) {
    await dingtalkApi.updateTaskStatus(taskId, isDone)
    const task = tasks.value.find(t => t.id === taskId)
    if (task) task.isDone = isDone
  }

  /** 全量同步 */
  async function fullSync() {
    loading.value.sync = true
    syncStatus.value = 'syncing'
    try {
      await dingtalkApi.fullSync()
      syncStatus.value = 'idle'
      lastSyncTime.value = Date.now()
      await fetchProjects()
      await fetchTasks()
    } catch {
      syncStatus.value = 'error'
    } finally {
      loading.value.sync = false
    }
  }

  /** 获取提醒设置 */
  async function fetchReminderSettings() {
    const res = await dingtalkApi.getReminderSettings()
    reminderSettings.value = res
  }

  /** 更新提醒设置 */
  async function updateReminderSettings(settings: Partial<ReminderSettings>) {
    const updated = {...reminderSettings.value, ...settings}
    await dingtalkApi.updateReminderSettings(updated)
    reminderSettings.value = updated
  }

  return {
    // State
    projects,
    currentProject,
    tasks,
    currentTask,
    syncStatus,
    lastSyncTime,
    reminderSettings,
    loading,
    // Getters
    todayTasks,
    tomorrowTasks,
    overdueTasks,
    todayCompletionRate,
    // Actions
    fetchProjects,
    createProject,
    fetchTasks,
    fetchTaskDetail,
    createTask,
    toggleTaskStatus,
    fullSync,
    fetchReminderSettings,
    updateReminderSettings,
  }
})
```

## 八、SQLite 本地缓存 - 必要性分析

### 8.1 当前项目数据流模式

```
┌─────────────┐     HTTP      ┌─────────────┐     HTTP      ┌─────────────┐
│  Vue 前端    │ ◀──────────▶ │  .NET Backend │ ◀──────────▶ │  外部 API   │
│  (Renderer) │   JSON/REST  │  (已构建)     │   业务处理    │  (钉钉)     │
└─────────────┘              └─────────────┘              └─────────────┘
        │
        │ IPC (部分功能)
        ▼
┌─────────────┐
│  SQLite     │  ← Electron 主进程本地存储
│  (Drizzle)  │
└─────────────┘
```

### 8.2 现有模式: work-collect

- **本地 SQLite**: 存储采集记录 (WorkRecord)
- **.NET Backend**: 提供 AI 分析 API
- **前端**: 通过 HTTP 调用 .NET API，通过 IPC 调用主进程读 SQLite

### 8.3 钉钉数据 - 两个方案

#### 方案 A: 后端主导 (推荐)

```
钉钉数据全部由 .NET Backend 管理和缓存
前端只通过 HTTP 调用 .NET API
SQLite 仅用于存储提醒设置
```

**优点**:
- 数据一致性由后端保证
- 减少 Electron 复杂度
- 与现有 work-collect 模式一致
- 离线可用性可由后端实现

**缺点**:
- 依赖后端服务在线

#### 方案 B: 混合模式

```
.NET Backend 调用钉钉 API
数据同步到 SQLite 本地缓存
前端可选择从 SQLite 或 .NET API 读取
```

**优点**:
- 支持离线场景
- 响应更快 (本地读取)

**缺点**:
- 数据同步复杂度高
- 可能出现数据不一致
- 增加 Electron 复杂度

### 8.4 结论

**推荐方案 A (后端主导)**:
- 钉钉任务数据由 .NET Backend 管理
- SQLite 仅存储提醒设置和历史记录
- 前端纯通过 HTTP 与 .NET Backend 交互
- 主进程定时器通过 .NET Backend API 获取数据

**是否需要 SQLite 缓存**:
- **任务数据**: 不需要，.NET Backend 负责缓存和同步
- **提醒设置**: 需要，存储在 Electron 本地
- **提醒历史**: 可选，可存储在本地或后端

## 十、注意事项
