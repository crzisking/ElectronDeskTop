# 技術選型文檔

## 1. 桌面框架：為什麼選 Electron？

市面上主流桌面框架有三個選擇：

| 維度 | **Electron** ✅ | Tauri | NW.js |
|------|----------------|-------|-------|
| 底層技術 | Node.js + Chromium | Rust + WebView | Node.js + Chromium |
| 安裝包大小 | 較大（~80MB） | 極小（~5MB） | 較大（~80MB） |
| Node.js 訪問 | 完整支持 | 受限（通過命令調用） | 完整支持 |
| 透明窗口 | 完整支持 ✅ | 部分平台有限制 | 支持 |
| 多窗口 | 原生支持 ✅ | 支持 | 支持 |
| 社區規模 | 極大（VSCode/Slack/Discord 都用它） | 成長中 | 較小 |
| 自動更新 | electron-updater 成熟 ✅ | 需自行集成 | 複雜 |

**選擇 Electron 的核心原因：**

- **浮球窗口**需要 `transparent: true` + `alwaysOnTop` 跨應用置頂，Electron 對此支持最成熟
- 需要本地文件讀寫、系統托盤、多窗口控制等桌面能力，Electron 主進程可直接使用 Node.js API
- `contextBridge`、`ipcMain/ipcRenderer`、`autoUpdater` 等關鍵能力都已非常成熟
- 前端開發者幾乎無需學習新語言，調試工具就是 Chromium DevTools

---

## 2. 前端框架：為什麼選 Vue 3？

| 維度 | **Vue 3** ✅ | React | Angular |
|------|-------------|-------|---------|
| TypeScript 支持 | 一等公民（Composition API 推導完整）| 需配置 | 原生支持 |
| 學習曲線 | 低 | 中 | 高 |
| 組件格式 | SFC（`.vue` 單文件組件） | JSX | 模板 |
| 配套 UI 庫 | Element Plus（功能最齊全） | Ant Design | NG-Zorro |
| 包體大小 | 小 | 中 | 大 |

**選擇 Vue 3 的理由：**

- Composition API 對 TypeScript 類型推斷支持最好，代碼可讀性也高
- Element Plus 是 Vue 生態最成熟的企業 UI 庫，表格、表單、彈窗、圖標均開箱即用
- SFC（`.vue`）文件格式把模板、邏輯、樣式放在一個文件裡，維護成本低

---

## 3. UI 組件庫：Element Plus

使用 Element Plus 的原因：

- Vue 3 官方推薦，原 Element UI 的 Vue 3 版本，API 穩定
- 組件齊全：`el-table`、`el-form`、`el-dialog`、`el-tabs`、`el-tag` 等企業常用組件全部有
- CSS 變量支持主題定制（修改 `--el-color-primary` 即可換主題色）
- 配合 `unplugin-vue-components` 可按需自動引入，無需手動 import 每個組件

---

## 4. 構建工具：electron-vite

**為什麼不直接用 Vite？**

Electron 應用有三個進程（main、preload、renderer），如果手動配置 Vite 需要維護 3 套配置文件，而且要手動處理 `__dirname` polyfill、預加載腳本重新構建觸發 Electron 重啟等問題。

`electron-vite` 一個配置文件搞定所有進程，HMR（渲染進程熱更新）開箱即用。

## 6. 構建命令

| 命令 | 說明 |
|------|------|
| `npm run dev` | 啟動開發模式（渲染進程熱更新，主進程自動重啟） |
| `npm run build` | 構建所有進程（輸出到 `out/`） |
| `npm run build:win` | 構建 + 打包 Windows 安裝包（NSIS + 便攜版） |
| `npm run build:mac` | 構建 + 打包 macOS DMG |
| `npm run build:linux` | 構建 + 打包 Linux AppImage |
| `npm run typecheck` | TypeScript 類型檢查（不含構建） |
| `npm run lint` | ESLint 代碼檢查與自動修復 |
| `npm run preview` | 用構建產物直接啟動 Electron（驗證構建結果） |

## 7. TypeScript 配置說明

項目有兩套 `tsconfig`，分別對應不同的進程環境：

- **`tsconfig.json`**：渲染進程（`src/`）+ 預加載腳本，瀏覽器環境，路徑別名 `@` → `src/`
- **`tsconfig.node.json`**：主進程（`electron/main/`），Node.js 環境，路徑別名 `@main` → `electron/main/`

兩套配置的必要性：主進程跑在 Node.js 環境，能使用 `fs`、`path` 等 Node API；渲染進程跑在 Chromium 沙盒裡，不能使用 Node API。如果用同一套 `tsconfig`，類型檢查就會亂。

## 8. 環境變量管理

所有 URL 地址不寫死在代碼裡，通過環境文件切換：

- `.env.development` — `npm run dev` 時自動生效，使用測試環境地址
- `.env.production` — `npm run build` 時自動生效，使用正式環境地址

只有 `VITE_` 前綴的變量才能在渲染進程（Vue 代碼）中訪問，其他變量只有 Node.js 進程可見。

當前的環境變量有：`VITE_API_BASE_URL`、`VITE_AUTH_BASE_URL`、`VITE_API_TIMEOUT`、`VITE_REPAIR_API_URL`、`VITE_AI_SOP_URL`、
`VITE_DIFY_URL`、`VITE_DIFY_API_KEY`（開發環境另有自動登錄用的 `VITE_DEV_AUTO_LOGIN`/`VITE_DEV_USERNAME`/
`VITE_DEV_PASSWORD`）。
