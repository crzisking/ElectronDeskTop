## FlowNodeData 重新設計（靜態業務流程留存）

### 背景

原有 `FlowNodeData` 包含 `status`、`estimatedDays`、`remark`、`assignee`、`department` 等動態欄位，設計前提是「追蹤流程執行狀態」。

但這個流程圖的實際用途是**靜態業務流程留存**：把業務流程以圖形化方式存入資料庫，方便日後根據流程圖查找對應的負責人。不需要也不應該有動態狀態欄位。

### 修改：FlowNodeData 欄位重構

```typescript
// 修改前
export interface FlowNodeData {
  label: string
  nodeType: FlowNodeType
  assignee?: string            // 移除
  department?: string          // 移除
  status?: 'pending' | 'in-progress' | 'completed' | 'rejected'  // 移除
  estimatedDays?: number       // 移除
  description?: string
  remark?: string              // 移除
}

// 修改後
export interface FlowNodeData {
  label: string
  nodeType: FlowNodeType
  employeeId?: string          // 新增：負責人工號（如 'A12345'）
  employeeName?: string        // 新增：負責人姓名
  departmentCode?: string      // 新增：部門代碼（如 'IT-001'）
  description?: string
}
```

### 修改文件清單

| 文件 | 修改內容 |
|------|---------|
| `src/types/api.types.ts` | `FlowNodeData` 移除 5 個舊欄位，新增 `employeeId`、`employeeName`、`departmentCode` |
| `src/components/VueFlowChart/nodes/TaskNode.vue` | 移除 `statusConfig` 計算屬性及狀態標籤；改顯示工號/姓名/部門代碼；新增 `description` 區塊 |
| `src/components/VueFlowChart/nodes/ApprovalNode.vue` | 同 TaskNode |
| `src/components/VueFlowChart/nodes/ConditionNode.vue` | 同上；保留左側和底部雙輸出 Handle |
| `src/components/VueFlowChart/nodes/StartNode.vue` | 精簡為只顯示標題和描述，移除無關欄位 |
| `src/components/VueFlowChart/nodes/EndNode.vue` | 同 StartNode |
| `src/components/VueFlowChart/VueFlowChart.vue` | `createNode()` 移除 `status: 'pending'` 默認值 |

## 節點雙擊編輯功能

### 背景

節點拖入畫布後，需要能填寫負責人資訊（工號、姓名、部門代碼）。使用雙擊節點開啟 `el-dialog` 的方式實現。

### 實作細節

**觸發：** Vue Flow 的 `@node-double-click` 事件，傳入 `{ node }` 參數

**編輯對話框表單欄位：**
- 節點標題（label）
- 工號（employeeId）
- 姓名（employeeName）
- 部門代碼（departmentCode）
- 描述（description，textarea，最多 200 字）

**數據寫回：** 確認後直接修改 `nodes.value` 中對應節點的 `data`，Vue Flow 響應式自動更新畫布

### 修改文件清單

| 文件 | 修改內容 |
|------|---------|
| `src/components/VueFlowChart/VueFlowChart.vue` | 新增 `editDialogVisible`、`editingNodeId`、`editForm` 響應式狀態；新增 `openEditDialog()`、`confirmEdit()`、`cancelEdit()` 方法；`useVueFlow()` 新增解構 `getNodes`；模板增加 `@node-double-click` 綁定和 `el-dialog` 編輯表單；左側面板底部增加操作提示（雙擊/Backspace/滾輪） |
| `src/views/Business/PipelineEditor.vue` | 工具欄提示文字增加「雙擊節點編輯資訊」 |

## 升級 Node.js 到 22

### 背景

項目之前鎖定在 Node 18（構建環境），系統已安裝 Node 22.12.0，升級以獲得更新的 JS 引擎特性和更長的 LTS 支持週期。

> 注意：Electron 28 內建 Node 版本仍為 20，這裡升級的是**構建工具鏈（electron-vite、vite、tsc）所使用的宿主 Node 版本**，不影響打包後 app 內的運行環境。

### 修改文件清單

| 文件 | 修改內容 |
|------|---------|
| `.nvmrc` | **新增**，內容 `22`，在項目目錄下執行 `nvm use` 自動讀取 |
| `package.json` | `@types/node` `^20.10.0` → `^22.0.0`；新增 `engines.node: ">=22.0.0"` |
| `tsconfig.json` | `target` / `lib`: `ES2020` → `ES2022`（渲染進程 + preload，跑在 Electron Chromium 環境） |
| `tsconfig.node.json` | `target` / `lib`: `ES2020` → `ES2023`（主進程，直接跑在 Node 22 上，可使用最新語法） |

### 驗證

```
node --version  → v22.12.0
npm run build   → ✓ 三個進程全部構建成功，零錯誤
```
