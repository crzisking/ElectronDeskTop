# 2026-04-08 開發記錄

## 新增功能：`electron-window` 第三種系統打開模式

### 背景

統一平台頁面原本只支持兩種系統打開方式：
- `iframe`：在主窗口內以 iframe 嵌入
- `external-browser`：用電腦默認瀏覽器打開

部分系統設置了 `X-Frame-Options` 禁止 iframe 嵌入，但用戶又不希望跳出應用到外部瀏覽器。因此新增第三種模式 `electron-window`：在 Electron 應用內新開一個獨立的 BrowserWindow 窗口加載網頁。

### 修改文件清單

| 文件 | 修改內容 |
|------|---------|
| `src/types/config.types.ts` | `SystemLink.openMode` 類型增加 `'electron-window'` 選項，並補充 JSDoc 說明 |
| `electron/shared/ipc-channels.ts` | 新增 `OPEN_CHILD_WINDOW: 'window:open-child'` IPC 頻道常量及完整文檔 |
| `electron/main/window-manager.ts` | 新增 `openChildWindow(url, title)` 方法，創建帶原生標題欄的子窗口 |
| `electron/main/ipc-handlers/index.ts` | 註冊 `OPEN_CHILD_WINDOW` 的 `ipcMain.handle` 處理器 |
| `electron/preload/index.ts` | 在 `window` 區塊新增 `openChild(url, title)` 方法，橋接到 IPC |
| `src/types/electron.d.ts` | 在 `window` 介面新增 `openChild` 類型聲明 |
| `src/views/UnifiedPlatform/UnifiedPlatformView.vue` | `handleOpenSystem()` 改為 switch-case，新增 `electron-window` 分支 |
| `src/views/UnifiedPlatform/SystemCard.vue` | openMode 標籤支持三種狀態：嵌入顯示(primary)、獨立窗口(warning)、外部瀏覽器(info) |
| `config/app-config.json` | 新增 `_help_openMode` 和 `_help_ssoEnabled` 欄位，說明 openMode 三種值的用途 |

### 完整 IPC 調用鏈（electron-window 模式）

```
用戶點擊系統卡片（openMode: 'electron-window'）
  → UnifiedPlatformView.vue: handleOpenSystem()
  → window.electronAPI.window.openChild(url, title)
  → preload/index.ts: ipcRenderer.invoke(IpcChannels.OPEN_CHILD_WINDOW, url, title)
  → ipc-handlers/index.ts: ipcMain.handle(OPEN_CHILD_WINDOW, ...)
  → window-manager.ts: openChildWindow(url, title)
  → 創建新 BrowserWindow，加載 URL
```

### 子窗口安全配置

```typescript
webPreferences: {
  contextIsolation: true,   // 渲染進程隔離
  nodeIntegration: false,   // 禁止 Node.js
  sandbox: true,            // Chromium 沙盒
}
```

子窗口使用原生標題欄（`frame: true`），不需要自定義標題欄和 preload 腳本。

## 為 app-config.json 全面添加注釋欄位

### 背景

JSON 格式不支持 `//` 或 `/* */` 注釋。為了讓開發人員在編輯配置時能快速理解每個欄位的用途和可選值，採用 `_comment` 前綴的欄位作為注釋。

### 注釋規範

- 文件開頭有一條全局說明：**所有以 `_comment` 開頭的欄位均為注釋，不參與程式邏輯**
- 每個配置區塊（`app`、`sidebar`、`floatingBall`、`unifiedPlatform`、`aiQuickFunctions`、`quickContact`）都有 `_comment` 說明該區塊用途
- 每個關鍵欄位都有對應的 `_comment_欄位名` 注釋，說明可選值、單位、用途

### 修改文件

| 文件 | 修改內容 |
|------|---------|
| `config/app-config.json` | 所有區塊和關鍵欄位添加 `_comment` 注釋欄位 |

### 各區塊注釋清單

| 區塊 | 注釋欄位數量 | 涵蓋內容 |
|------|-------------|---------|
| 頂層 | 1 | 全局說明 `_comment` 欄位的性質 |
| `app` | 5 | language 可選值、theme 可選值、startMinimized、launchOnStartup |
| `sidebar` | 3 | defaultCollapsed、items 陣列結構說明 |
| `floatingBall` | 7 | size、opacity、defaultPosition、snapToEdge、quickMenu 結構、action.type 所有可選值 |
| `unifiedPlatform` | 6 | openMode 三種值詳細說明、ssoEnabled、ssoTokenParam、systems 結構 |
| `aiQuickFunctions` | 4 | apiBaseUrl、apiTimeout 單位、tools 結構 |
| `quickContact` | 4 | searchApiEndpoint、emailApiEndpoint、maxSearchResults |
